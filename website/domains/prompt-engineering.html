Prompt Engineering

How to write instructions that get reliable, accurate responses from LLMs — the most practical skill in the exam.

Why This Domain Matters

An LLM is only as good as the instructions you give it. Prompt engineering is the skill of crafting inputs that reliably produce the output you want. In agentic systems, every single LLM call has a prompt — and a bad prompt means a broken agent. This domain is ~10% of the exam but 100% of day-to-day AI development.

Exam tip: The exam tests practical application — "which prompting technique would you use for X?" Know zero-shot, few-shot, chain-of-thought, and ReAct prompting cold.

Prompting Techniques Cheat Sheet

TechniqueWhen to UseExample
Zero-shot Simple, well-defined tasks "Classify this text as positive or negative: ..."
Few-shot Specific output format needed Give 2-3 examples before the real query
Chain-of-Thought Multi-step reasoning, math, logic "Think step by step before giving your answer."
ReAct Agent loops with tool use Thought → Action → Observation format
Self-Consistency High-stakes decisions Run prompt N times, pick majority answer
Role Prompting Domain-specific answers "You are an expert Python developer. ..."

Exam Tips

Common exam trap: Chain-of-Thought only improves reasoning in models large enough to follow it (usually 7B+ parameters). Applying CoT to tiny models can hurt performance.
  • Know the difference between zero-shot and few-shot — and when few-shot is worth the extra tokens
  • Know what a system prompt is and what it controls
  • Know what prompt injection is — and that NeMo Guardrails defends against it
  • Know that ReAct prompting is what makes the Thought→Action→Observation loop work
  • Know that output format instructions (JSON, markdown) in the prompt improve parseability
