Tool Use & Function Calling

How LLMs call external functions, APIs, and services — what makes an agent actually useful.

Why This Domain Matters

Without tool use, an LLM can only work with the information in its training data. Function calling lets the LLM reach out to the real world — search the web, query databases, run code, call APIs. This is what separates a chatbot from an agent. The exam heavily tests your understanding of how tool schemas work and how NVIDIA NIMs support function calling.

NVIDIA focus: Know how to define a tool for NVIDIA NIM specifically (OpenAI-compatible JSON schema). Run the NVIDIA GenerativeAI Examples Tool Calling notebook before the exam.

NVIDIA NIMs Hands-On

Tool JSON Schema (Know This Cold)

{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "Get current weather for a city",
    "parameters": {
      "type": "object",
      "properties": {
        "city": {
          "type": "string",
          "description": "The city name, e.g. 'San Francisco'"
        },
        "unit": {
          "type": "string",
          "enum": ["celsius", "fahrenheit"],
          "description": "Temperature unit"
        }
      },
      "required": ["city"]
    }
  }
}

This exact format is what you pass to NVIDIA NIMs, OpenAI, and LangChain. Know it by heart.

Exam Tips

Common exam trap: The LLM does NOT execute the tool — it only produces a JSON call. YOUR code executes the tool and passes the result back to the LLM. The LLM never runs code directly.
  • Know the JSON schema structure for function calling (name, description, parameters)
  • Know required fields vs optional fields in the schema
  • Know that tool_choice="auto" lets the LLM decide whether to call a tool
  • Know how to handle tool errors — pass the error back as the tool result
  • Know that NVIDIA NIMs use the same OpenAI-compatible format
  • Know what parallel tool calling is — multiple tools called in one LLM step
