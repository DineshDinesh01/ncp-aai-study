Memory & RAG

How agents remember things — from short-term context to long-term document knowledge via Retrieval-Augmented Generation.

Why This Domain Matters

LLMs have two fundamental memory problems: (1) they forget everything when the context window ends, and (2) they don't know about your private data. RAG solves the second problem — it lets an agent retrieve relevant information from your documents at query time. Memory systems solve the first — they persist state across conversations.

Exam focus: Know the full RAG pipeline cold. The exam will ask about chunk size, embedding models, cosine similarity, and how to improve retrieval quality. Know NVIDIA's NeMo Retriever.

The 4 Types of Agent Memory

In-context memory Everything in the current prompt Pass full conversation history Context window size

Exam Tips

Common exam trap: "Hallucination" in RAG usually means the retrieval step failed, not the generation step. Always diagnose retrieval quality first — check chunk size, embedding model quality, and top-k value.
  • Know the difference between sparse retrieval (BM25/keyword) and dense retrieval (embeddings)
  • Know what chunk size does — smaller = precise but may lose context; larger = more context but noisier
  • Know what cosine similarity measures (angle between vectors, 0-1)
  • Know HNSW index — hierarchical navigable small world, used for fast ANN search
  • Know NeMo Retriever is NVIDIA's production RAG service
  • Know what hybrid search is — combines BM25 + dense retrieval
