Agentic System Design

How to design AI systems that reason, plan, and act autonomously — the core of this certification.

Why This Is the Most Important Domain

This domain carries ~20% of the exam — more than any other. An agent is an AI system that doesn't just answer questions — it takes actions, uses tools, checks results, and keeps going until the task is done. You need to understand how agents are structured, how they make decisions, and how to implement them with LangGraph.

Priority: If you only have time to go deep on two domains, make this one of them. The exam will have scenario-based questions like "You need to build an agent that does X — which architecture do you use?"

The Core Agent Loop

Every agent follows the same fundamental pattern, called the ReAct loop:

ThoughtLLM reasons about the task
→
ActionLLM picks a tool + args
→
ObservationTool result returned
→
Repeat or AnswerKeep going or finish

Exam Scenario Practice

Scenario: You need to build a research agent that: (1) searches the web for information, (2) checks if it found enough, (3) if not, refines the search and tries again (max 3 times), (4) synthesizes a final answer. Which framework? How do you implement the retry loop?

Answer: Use LangGraph. Create a StateGraph with: a search node, a check_results node with conditional edge (if enough → synthesize, else → search again), and a counter in state to enforce the max 3 limit. ReAct can't enforce the retry limit without extra code — LangGraph makes it explicit.

Key Terms to Know

  • Agent — An LLM + tools + a loop that keeps running until done
  • Node — A function in a LangGraph graph (processes state, returns updates)
  • Edge — A connection between nodes (can be conditional)
  • State — A TypedDict that flows through the graph and accumulates results
  • Checkpoint — A snapshot of state saved to memory (enables session persistence)
  • Human-in-the-loop — An interrupt that pauses execution for human approval
  • Scratchpad — The intermediate reasoning steps in a ReAct agent's prompt
