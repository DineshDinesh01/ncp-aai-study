LLM Fundamentals

How large language models work under the hood — the essential foundation for everything else in this certification.

Why This Domain Matters

Every agentic system is built on top of an LLM. If you don't understand how the model works, you can't debug it, optimize it, or use it correctly. This domain covers the internal machinery — transformers, tokenization, embeddings, and fine-tuning — that every other domain depends on.

Exam tip: This domain is ~15% of the exam but it is a prerequisite for understanding everything else. If you skip it, the later domains won't make sense. Study this first.

Exam Tips for This Domain

Common exam trap: Know the difference between encoder-only (BERT — classification), decoder-only (GPT — generation), and encoder-decoder (T5 — translation/summarization) architectures. The exam will ask which to use for a given task.
  • Know what self-attention is and why it replaces RNNs
  • Know what positional encoding does (model has no inherent sequence order)
  • Know what BPE tokenization is and why different models tokenize differently
  • Know that LoRA injects trainable low-rank matrices — doesn't retrain all weights
  • Know NeMo Toolkit is NVIDIA's primary tool for fine-tuning
  • Understand what a context window is and why token count matters
