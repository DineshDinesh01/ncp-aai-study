LLM Optimization Techniques

TechniqueWhat It DoesTradeoff
INT8 Quantization Reduces weights from FP32 to INT8 ~2x memory reduction, slight accuracy loss
INT4 Quantization Reduces weights to 4-bit ~4x memory reduction, more accuracy loss
Tensor Parallelism Splits model layers across multiple GPUs Requires multiple GPUs, communication overhead
Dynamic Batching Groups multiple requests into one batch Higher throughput, slight latency increase
Paged KV Cache Efficient attention cache memory management Higher throughput, more concurrent requests
In-flight Batching Adds new requests to ongoing batch GPU utilization maximized, complex scheduling

Exam Tips

Common exam trap: Know which tool does what. TensorRT-LLM optimizes the model. Triton serves the model. NIMs are pre-built packages that include both. These are 3 different things.
  • Know Triton's 3 ports: 8000 (HTTP), 8001 (gRPC), 8002 (metrics)
  • Know the model repository format for Triton
  • Know what dynamic batching does in Triton
  • Know that TensorRT-LLM uses paged KV cache for memory efficiency
  • Know that NIMs expose an OpenAI-compatible API
  • Know the latency vs throughput tradeoff in batching
