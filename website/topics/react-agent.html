ReAct Agent Loop

ReAct (Reasoning + Acting) is the core pattern for agentic AI. The agent interleaves reasoning (Thought) with tool execution (Action) and observes results — looping until it can answer. This is the exam's most tested agentic pattern.

1Definition

ReAct (Yao et al., 2022) is an agent prompting strategy that combines Reasoning (producing a Thought step) with Acting (executing a tool call). The LLM produces interleaved Thought → Action → Observation triplets in a loop until it has enough information to produce a final answer.

Unlike pure chain-of-thought (reasoning only) or pure tool use (acting only), ReAct does both simultaneously — the agent reasons about what it needs, takes action to get it, observes the result, and reasons again.

2Why It Exists

Problem with pure CoT: Chain-of-thought reasoning is powerful but the LLM can only reason about what it knows at training time. It hallucinates when it needs external information.

Problem with pure tool use: Without reasoning, the agent blindly calls tools without explaining why or checking if the result is reasonable.

ReAct's solution: The agent reasons ("I need the current stock price") before acting ("Call get_stock_price"), then reasons about what it got ("The price is $134, now I can answer"). This produces more reliable, auditable agent behavior.

3How It Works

The ReAct loop in action — here's what the LLM actually outputs in the scratchpad:

textReAct Scratchpad Example
User: What is NVIDIA's latest GPU and how many CUDA cores does it have?

Thought: I need to find NVIDIA's latest GPU. I'll search for it.
Action: search
Action Input: NVIDIA latest GPU 2024 2025
Observation: NVIDIA's latest GPU is the H200, announced in 2023 with 16,896 CUDA cores...

Thought: I found the GPU name and CUDA core count. I can now answer.
Final Answer: NVIDIA's latest GPU is the H200, which has 16,896 CUDA cores.

The loop has a maximum iteration limit (default 10 in LangChain). If the agent doesn't reach a "Final Answer" within this limit, it returns an error or partial answer.

5Implementation

pythonreact_agent.py
# pip install langchain-community langchain-openai duckduckgo-search
from langchain.agents import AgentExecutor, create_react_agent
from langchain_community.tools import DuckDuckGoSearchRun
from langchain_openai import ChatOpenAI
from langchain_core.prompts import PromptTemplate

# Tools
tools = [DuckDuckGoSearchRun(name="search")]

# LLM (replace with NVIDIA NIM for free usage)
llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0)

# ReAct prompt template
react_prompt = PromptTemplate.from_template("""
You are a helpful research assistant with access to the following tools:

{tools}

Tool Names: {tool_names}

To answer the question, use this exact format:
Thought: What do I need to find out?
Action: tool_name
Action Input: what to pass to the tool
Observation: result from the tool
... (repeat Thought/Action/Observation as needed)
Thought: I now have enough information
Final Answer: the complete answer

Question: {input}
{agent_scratchpad}
""")

# Build agent + executor
agent = create_react_agent(llm, tools, react_prompt)
executor = AgentExecutor(
    agent=agent,
    tools=tools,
    verbose=True,          # shows the full reasoning trace
    max_iterations=7,      # prevent infinite loops
    handle_parsing_errors=True  # graceful error recovery
)

# Run it
result = executor.invoke({
    "input": "What is NVIDIA's NeMo Guardrails and what language does it use for dialogue flows?"
})
print(f"\nFinal Answer: {result['output']}")

7Hands-On Lab

Objective: Build a research agent that traces its full reasoning. Ask it a question that requires 2+ tool calls.

Steps: 1. Install dependencies. 2. Copy Section 5 code. 3. Run with verbose=True. 4. Ask: "What is NVIDIA's latest LLM model and what context length does it support?" 5. Read the scratchpad output — identify each Thought, Action, Observation.

Challenge: Add a second tool (calculator) and ask: "If an H100 GPU costs $30,000 and I need 8 of them, what is the total cost in USD and EUR at 1.08 exchange rate?"

Verify: The agent should call search once, then calculator once, then give the final answer. The scratchpad should show exactly 2 tool calls.

8Common Mistakes

❌ Mistake 1 — No max_iterations

Without a limit, a confused agent loops forever burning API credits. Fix: Always set max_iterations=5 to 10.

❌ Mistake 2 — Tool Output Too Long

A search result might be 10,000 tokens. Passing it all to the LLM wastes context and may truncate the rest of the conversation. Fix: Truncate tool output to ~1500 characters before returning it as an observation.

❌ Mistake 3 — Wrong Prompt Format

The ReAct prompt requires exact formatting (Thought:/Action:/Action Input:/Observation:). Any deviation confuses the output parser. Fix: Use handle_parsing_errors=True and the LangChain ReAct prompt template exactly as provided.

9Exam Relevance

🎓 Exam Domain: Agentic System Design (~20% of exam)

Key terms: ReAct, Thought-Action-Observation, scratchpad, max_iterations, AgentExecutor, tool routing, handle_parsing_errors, grounding, hallucination reduction

Sample question: "In the ReAct framework, what is the correct sequence of outputs produced by the LLM?"
Answer: Thought → Action → (your code runs tool) → Observation → repeat → Final Answer

Sample question 2: "An agent is stuck in a loop calling the same tool repeatedly. What should you check first?"
Answer: Check max_iterations limit and whether handle_parsing_errors is enabled. Also verify the tool description is clear enough for the LLM to know when to stop.
