Tool Use & Function Calling

Function calling lets an LLM invoke external tools — APIs, databases, calculators, code executors — by outputting structured JSON that your code parses and executes. This is how agents gain real-world capabilities.

1Definition

Function calling (tool use) is a capability that allows LLMs to signal that they want to call a specific function with specific arguments, rather than answering in plain text. The LLM outputs a structured JSON object; your application code receives it, executes the corresponding function, and returns the result back to the LLM.

Example: Instead of making up a stock price, the LLM outputs {"name": "get_stock_price", "arguments": {"ticker": "NVDA"}}. Your code calls the real API, gets the real price, and feeds it back to the LLM for the final answer.

2Why It Exists

Problem: LLMs have a knowledge cutoff, can't do precise arithmetic, can't access live data, and can't modify external systems. A pure text-generation LLM is fundamentally limited as an agent.

Solution: Function calling turns the LLM into a controller — it decides WHAT action to take and with WHAT parameters. Your application executes the action. The LLM synthesizes the result into a natural language response.

This is what separates a chatbot from an agent: agents take actions in the world.

3How It Works

Step 1 — Define tools as JSON schemas: Each tool has a name, description, and parameter schema. The LLM receives this as part of its context.

Step 2 — LLM receives user message + tool list

Step 3 — LLM decides to call a tool: Output is a tool_call object with function name + JSON arguments (not regular text)

Step 4 — Your code executes the function and gets a real result

Step 5 — Result injected as tool message: The LLM receives the result and generates the final user-facing answer

jsonTool Schema Structure
{
  "type": "function",
  "function": {
    "name": "get_weather",
    "description": "Get current weather for a city. Use this when the user asks about weather.",
    "parameters": {
      "type": "object",
      "properties": {
        "city": {
          "type": "string",
          "description": "The city name, e.g. 'San Jose'"
        },
        "unit": {
          "type": "string",
          "enum": ["celsius", "fahrenheit"],
          "description": "Temperature unit"
        }
      },
      "required": ["city"]
    }
  }
}

5Implementation

pythontool_calling_agent.py
# pip install langchain-community langchain-openai duckduckgo-search
from langchain.agents import AgentExecutor, create_react_agent
from langchain_community.tools import DuckDuckGoSearchRun
from langchain.tools import StructuredTool
from langchain_core.prompts import PromptTemplate
from langchain_openai import ChatOpenAI
from pydantic import BaseModel
import math

# ── Define custom tools ──────────────────────────────────

class CalcInput(BaseModel):
    expression: str

def calculator(expression: str) -> str:
    """Safely evaluate a math expression."""
    try:
        # Safe eval: only allow math operations
        allowed = {k: v for k, v in math.__dict__.items() if not k.startswith("_")}
        result = eval(expression, {"__builtins__": {}}, allowed)
        return str(result)
    except Exception as e:
        return f"Error: {e}"

calc_tool = StructuredTool.from_function(
    func=calculator,
    name="calculator",
    description="Evaluate math expressions. Input: a string like '2 * 3.14 * 5'",
    args_schema=CalcInput
)

search_tool = DuckDuckGoSearchRun()
tools = [search_tool, calc_tool]

# ── Set up LLM (use NVIDIA NIM endpoint for free) ────────
# Replace with: base_url="https://integrate.api.nvidia.com/v1"
llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0)

# ── ReAct prompt ─────────────────────────────────────────
prompt = PromptTemplate.from_template("""
Answer the question using the available tools.

Tools available:
{tools}

Tool names: {tool_names}

Format:
Thought: [your reasoning]
Action: [tool name]
Action Input: [tool input]
Observation: [tool result]
... (repeat as needed)
Thought: I now have the final answer
Final Answer: [your answer]

Question: {input}
{agent_scratchpad}
""")

# ── Build and run agent ──────────────────────────────────
agent = create_react_agent(llm, tools, prompt)
executor = AgentExecutor(agent=agent, tools=tools, verbose=True, max_iterations=5)

result = executor.invoke({"input": "What is NVIDIA's stock ticker, and what is 2 * pi * 76?"})
print(f"\nFinal Answer: {result['output']}")

pythonnim_tool_calling.py
# Direct NIM function calling (build.nvidia.com — free credits)
from openai import OpenAI

client = OpenAI(
    base_url="https://integrate.api.nvidia.com/v1",
    api_key="nvapi-YOUR_FREE_KEY"
)

tools = [{
    "type": "function",
    "function": {
        "name": "get_gpu_specs",
        "description": "Get technical specifications for an NVIDIA GPU model",
        "parameters": {
            "type": "object",
            "properties": {
                "model": {"type": "string", "description": "GPU model name e.g. 'H100'"},
                "spec": {"type": "string", "enum": ["memory", "cuda_cores", "tdp"]}
            },
            "required": ["model"]
        }
    }
}]

response = client.chat.completions.create(
    model="meta/llama-3.1-70b-instruct",
    messages=[{"role": "user", "content": "What memory does the H100 GPU have?"}],
    tools=tools,
    tool_choice="auto"
)

tool_call = response.choices[0].message.tool_calls[0]
print(f"Tool called: {tool_call.function.name}")
print(f"Arguments: {tool_call.function.arguments}")

8Common Mistakes

❌ Mistake 1 — Vague Tool Descriptions

The LLM decides WHICH tool to call based on the description. A vague description like "does stuff" means the LLM won't know when to call it. Fix: Write descriptions that tell the LLM exactly WHEN to use this tool, e.g. "Use this when the user asks about current weather."

❌ Mistake 2 — No max_iterations Limit

An agent without a max_iterations cap can loop forever if it keeps making wrong tool calls. Fix: Always set max_iterations=5 to 10. Handle the "reached max iterations" case gracefully.

❌ Mistake 3 — Tool Output Too Large for Context

If a tool returns 50,000 tokens (e.g., a full web page), it exceeds the context window. Fix: Truncate tool output before returning it to the LLM. Common limit: 2000 characters per tool result.
