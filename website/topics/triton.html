1What Is Triton?

NVIDIA Triton Inference Server is an open-source, production-grade inference serving platform that can host any AI model on any NVIDIA GPU hardware. It supports simultaneous serving of multiple models across multiple frameworks (PyTorch, TensorRT, ONNX, vLLM, TensorRT-LLM) through a unified API.

2Why Triton Exists

Deploying models directly via Python inference loops does not scale. Triton provides: dynamic batching (combines multiple requests into efficient GPU batches), concurrent model execution (multiple model instances on one GPU), model versioning (blue/green deployments), and ensemble models (chain preprocessing + model + postprocessing into a single API call).

3Model Repository Layout

Triton loads models from a model repository — a directory with a specific layout:

Shell — Triton Model Repository Structure

model_repository/
├── my_llm/                    # model name (matches API endpoint)
│   ├── config.pbtxt           # REQUIRED: model configuration
│   ├── 1/                     # version 1 (numerically named)
│   │   └── model.plan         # TensorRT engine / onnx / weights
│   └── 2/                     # version 2 (for blue/green rollout)
│       └── model.plan
└── my_preprocessor/
    ├── config.pbtxt
    └── 1/
        └── model.py           # Python backend
Protobuf — config.pbtxt Example

name: "my_llm"
backend: "tensorrtllm"          # or "onnxruntime", "pytorch", "vllm"
max_batch_size: 8

input [
  { name: "input_ids"    data_type: TYPE_INT32  dims: [-1] },
  { name: "attention_mask" data_type: TYPE_INT32 dims: [-1] }
]
output [
  { name: "logits"  data_type: TYPE_FP32  dims: [-1, 32000] }
]

dynamic_batching {
  preferred_batch_size: [4, 8]
  max_queue_delay_microseconds: 1000
}

instance_group [
  { count: 2  kind: KIND_GPU  gpus: [0, 1] }     # 2 instances on GPUs 0,1
]
4Starting Triton & Making Requests
Shell — Launch Triton

docker run --gpus all -p 8000:8000 -p 8001:8001 -p 8002:8002 \
  -v /path/to/model_repository:/models \
  nvcr.io/nvidia/tritonserver:25.01-py3 \
  tritonserver --model-repository=/models
Python — gRPC Client (Preferred)

import tritonclient.grpc as grpcclient
import numpy as np

client = grpcclient.InferenceServerClient(url="localhost:8001")

# Check server health
print(client.is_server_ready())  # True

# Prepare input
input_ids = np.array([[101, 2054, 2003, 2115, 102]], dtype=np.int32)
inputs = [grpcclient.InferInput("input_ids", input_ids.shape, "INT32")]
inputs[0].set_data_from_numpy(input_ids)

# Infer
outputs = [grpcclient.InferRequestedOutput("logits")]
result = client.infer("my_llm", inputs, outputs=outputs)
logits = result.as_numpy("logits")
5perf_analyzer Benchmarking

perf_analyzer is Triton's official benchmarking CLI tool — measures latency and throughput under various load conditions.

Shell — perf_analyzer Usage

# Concurrency mode: measure at 8 concurrent requests
perf_analyzer -m my_llm -u localhost:8001 --protocol grpc \
  --concurrency-range 1:16:2 \
  --measurement-interval 5000 \
  --output-shared-memory none

# Output includes:
# Concurrency: 8
# Throughput: 452.3 infer/sec
# p50 latency: 17.6 ms
# p95 latency: 23.1 ms
# p99 latency: 28.4 ms
6NVIDIA Ecosystem — Triton + TensorRT-LLM

In the NVIDIA production stack, Triton is the serving frontend and TensorRT-LLM is the backend engine. They are deployed together: TensorRT-LLM handles model optimization (quantization, kernel fusion), Triton handles request routing, batching, and autoscaling.

NVIDIA NIMs are pre-packaged containers that bundle Triton + TensorRT-LLM + model weights into a single deployable unit with an OpenAI-compatible REST API — you get Triton's performance without manually configuring it.

8Common Mistakes
❌ Using HTTP When gRPC Is Available

gRPC (port 8001) is significantly faster than HTTP (8000) for production — binary serialization, multiplexing, bi-directional streaming. Always prefer gRPC in production agents.

❌ Wrong Port in Agent Code

A common exam trick: agents hang or timeout because they call port 8000 (HTTP) but have gRPC configured, or vice versa. Match client protocol to server configuration.

❌ Missing config.pbtxt

Triton refuses to load any model without a valid config.pbtxt. Even if you use auto-completion, explicitly specifying inputs/outputs is required for most backends.
