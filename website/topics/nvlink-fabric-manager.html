1. NVLink Technology

NVLink is NVIDIA's proprietary high-speed serial interconnect between GPUs. Unlike PCIe (which routes GPU traffic through the CPU), NVLink enables direct GPU-to-GPU communication at up to 3× PCIe bandwidth on Gen 5.

NVLink GenGPULinks per GPUBW per Link (bidir)Total GPU BW
NVLink 3A1001250 GB/s600 GB/s
NVLink 4H100, H2001850 GB/s900 GB/s
NVLink 5B200, GB20018100 GB/s1.8 TB/s

2. NVSwitch Fabric

NVSwitch is the dedicated switching ASIC that creates the all-to-all non-blocking fabric connecting all GPUs inside a DGX (or across a multi-chassis system in GB200 NVL72).

NVSwitch Topology in DGX H100
4 NVSwitch chips × 18 ports each = every GPU has 4 NVLink paths to every other GPU, providing full bandwidth in all directions simultaneously. No GPU needs to share NVLink bandwidth with another GPU for non-competing operations. 
# See NVSwitch in topology matrix
nvidia-smi topo -m

# Output legend:
# NV1  = Connected via 1 NVLink
# NV2  = Connected via 2 NVLinks
# NV4  = Connected via 4 NVLinks (typical DGX H100 GPU-to-GPU)
# SYS  = Connected through CPU QPI/PCIe only (different NUMA domain)
# PHB  = Connected through PCIe host bridge

3. Fabric Manager

The NVIDIA Fabric Manager (nv-fabricmanager) is a daemon that programs the NVSwitch routing tables, enabling multi-GPU NVLink communication. On DGX systems with NVSwitches, it must be running for P2P NVLink transfers to work.

# Service management
sudo systemctl start nvidia-fabricmanager
sudo systemctl stop nvidia-fabricmanager
sudo systemctl status nvidia-fabricmanager
sudo systemctl enable nvidia-fabricmanager   # Start on boot

# Check Fabric Manager logs
journalctl -u nvidia-fabricmanager -n 50 --no-pager
# OR
tail -f /var/log/fabricmanager.log

# Verify FM version
nv-fabricmanager -v

# Check if FM is active and GPUs are registered
nv-fabricmanager --get-status
Critical: FM Must Match Driver Version
Fabric Manager version must exactly match the CUDA driver version (e.g., both 550.54.15). A mismatch will prevent FM from starting, and NVLink will not activate. After driver upgrades, always update Fabric Manager to the same version.

4. Verification Commands

# Full NVLink status per GPU
nvidia-smi nvlink -s            # Status: Active/Inactive per link

# NVLink speed per GPU (bandwidth per link)
nvidia-smi nvlink -i 0 -s

# NVLink error counters
nvidia-smi nvlink -e            # Error counts for all GPUs
nvidia-smi nvlink -i 0 -e      # GPU 0 only

# Clear NVLink error counters (after investigation)
nvidia-smi nvlink -i 0 -r      # Reset counters GPU 0

# Confirm all 18 links active (H100 example)
nvidia-smi nvlink -s | grep "Link"
# Expected: Link 0: Active, Link 1: Active, ... Link 17: Active

# P2P access matrix (shows which GPUs can directly P2P copy)
nvidia-smi topo -p2p r          # Read P2P matrix
nvidia-smi topo -p2p w          # Write P2P matrix

# Typical NVLink troubleshooting sequence:
# 1. Check FM is running
systemctl status nvidia-fabricmanager

# 2. Check all GPUs visible
nvidia-smi -L

# 3. Check NVLink status
nvidia-smi nvlink -s

# 4. Check for errors
nvidia-smi nvlink -e

# 5. If FM fails — check version match
nvidia-smi --query-gpu=driver_version --format=csv,noheader
nv-fabricmanager -v

# 6. Restart FM if needed
sudo systemctl restart nvidia-fabricmanager

7. Exam Prep & Quick Reference

Must-Know for Exam
  • Fabric Manager is REQUIRED for NVSwitch-based multi-GPU NVLink (DGX A100, H100, H200, B200)
  • FM version must match driver version exactly
  • NVLink 3 (A100): 600 GB/s | NVLink 4 (H100/H200): 900 GB/s | NVLink 5 (B200): 1.8 TB/s
  • nvidia-smi nvlink -s shows Active/Inactive per link; should show all links Active
  • nvidia-smi nvlink -e shows error counters; growing counters = hardware fault risk
  • nvidia-smi topo -m matrix shows NV4 for NVLink connections (4 paths per pair on DGX H100)
  • NVLink does not use TCP/IP — it is a direct chip-to-chip interconnect
