Human-AI Interaction & HITL Oversight

Building agents that know when to ask for human approval — LangGraph interrupt() + Command pattern, checkpointing with MemorySaver/SqliteSaver, approval gates, escalation paths, time-travel debugging, and audit trails.

1What Is Human-in-the-Loop (HITL)?

Human-in-the-Loop (HITL) is an architectural pattern where an AI agent pauses its execution at a defined checkpoint, presents its planned action (or a question) to a human, and waits for explicit approval or correction before continuing. This is critical for high-stakes actions: sending emails, executing financial transactions, modifying production databases, generating reports that go to external clients.

LangGraph provides first-class HITL support via the interrupt() function introduced in LangGraph 0.2.x and made part of LangGraph Platform GA at the "Interrupt 2025" conference.

2LangGraph interrupt() + Command Pattern
Python — LangGraph HITL with interrupt()

from langgraph.graph import StateGraph, END
from langgraph.checkpoint.memory import MemorySaver
from langgraph.types import interrupt, Command
from typing import TypedDict

class State(TypedDict):
    draft_email: str
    approved: bool
    user_feedback: str

def draft_agent(state: State) -> State:
    """Agent drafts an email based on user request."""
    return {"draft_email": "Dear Client, I am pleased to inform you..."}

def human_review_node(state: State) -> Command:
    """Pause here and wait for human to review the draft email."""
    human_decision = interrupt({
        "draft_email": state["draft_email"],
        "instructions": "Please review this draft email. Reply 'approve' or provide feedback."
    })
    # human_decision is what the human provided via Command(resume=...)
    if human_decision.lower() == "approve":
        return Command(goto="send_email", update={"approved": True})
    else:
        return Command(goto="draft_agent", update={
            "approved": False,
            "user_feedback": human_decision
        })

def send_email(state: State) -> State:
    """Only reached after human approval."""
    print(f"Sending approved email: {state['draft_email']}")
    return state

# Build graph with checkpointing
checkpointer = MemorySaver()
builder = StateGraph(State)
builder.add_node("draft_agent", draft_agent)
builder.add_node("human_review", human_review_node)
builder.add_node("send_email", send_email)
builder.set_entry_point("draft_agent")
builder.add_edge("draft_agent", "human_review")
builder.add_edge("send_email", END)
graph = builder.compile(checkpointer=checkpointer)

# Run until interrupt
config = {"configurable": {"thread_id": "email-task-001"}}
result = graph.invoke({"draft_email": ""}, config=config)
# → Graph pauses at interrupt() in human_review_node

# Human reviews and resumes
graph.invoke(Command(resume="approve"), config=config)
# → Graph continues to send_email node
3Checkpointers — State Persistence

Checkpointers save the agent's full state at every step. This enables: HITL (pause and resume), time-travel debugging (replay from any past state), and conversational memory across sessions.

CheckpointerStorageBest For
MemorySaverIn-memory dictionaryDevelopment, testing, single-session agents
SqliteSaverSQLite fileSingle-server production, persistent across restarts
PostgresSaverPostgreSQL databaseMulti-server production, high availability
AsyncPostgresSaverPostgreSQL (async)High-throughput async LangGraph deployments
4Time-Travel Debugging

Because checkpointers store every state, you can "time-travel" — inspect the agent's full state at any past step, replay from that point with a different input, or compare two execution paths. This is essential for debugging complex agentic failures.

Python — Time-Travel with LangGraph

# Get full execution history
history = list(graph.get_state_history(config))
for state in history:
    print(f"Step: {state.next}, Values: {state.values}")

# Replay from a specific checkpoint
past_config = history[2].config   # step 2
graph.invoke(Command(resume="approve"), config=past_config)
5Exam Angles
🎯 Exam Key Points — Human-AI Interaction (5%)
  • interrupt() pauses graph execution at a node; Command(resume=...) continues it
  • MemorySaver: in-memory (dev). SqliteSaver: single-server. PostgresSaver: multi-server production
  • Thread ID (thread_id) in configurable identifies a specific conversation/task session
  • Time-travel = replay from any historical checkpoint using get_state_history()
  • HITL use cases: email approval, financial transaction confirmation, external report generation
  • LangGraph Platform GA announced at "Interrupt 2025" conference
