1. What Is DCGM?

Data Center GPU Manager (DCGM) is NVIDIA's lightweight agent for cluster-scale GPU monitoring, health checking, policy enforcement, and diagnostics. It runs as a host daemon (nv-hostengine) and is controlled via the dcgmi CLI or REST API.

DCGM vs nvidia-smi
  • nvidia-smi: Single-node, per-process view. Interactive and ad-hoc.
  • DCGM: Cluster-aware, group-based monitoring. Integrates with Prometheus, Kubernetes (DCGM Exporter), and Slurm. Better for production observability.
# Start the DCGM host engine
sudo systemctl start nvidia-dcgm      # systemd service
# OR standalone:
sudo nv-hostengine

# Verify DCGM is running
dcgmi discovery -l                    # Should list all GPUs

2. Discovery & Groups

# Discovery
dcgmi discovery -l                   # List all detected GPUs
dcgmi discovery -i 0                 # Detailed info for GPU 0

# Groups — manage sets of GPUs together
dcgmi group -l                       # List all groups
dcgmi group -c mygroup               # Create group named 'mygroup'
dcgmi group -a -g 1 -i 0,1,2,3      # Add GPU 0-3 to group ID 1
dcgmi group -r -g 1 -i 2            # Remove GPU 2 from group 1
dcgmi group -d -g 1                  # Delete group 1
dcgmi group --default                # Use built-in 'all GPUs' group (ID 1)

3. Health Monitoring

# Check health of all GPUs
dcgmi health -g 1 -c         # Set health watches on group 1
dcgmi health -g 1 -e         # Check (evaluate) health on group 1

# Health subsystems checked:
# - pcie:    PCIe bus errors
# - nvlink:  NVLink errors
# - ecc:     Memory ECC errors
# - power:   Power limit violations
# - thermal: Temperature threshold violations
# - mem:     Memory bandwidth errors

# Example: check individual subsystems
dcgmi health -g 1 -e --pcie --nvlink --ecc
Health ResultMeaningAction
HealthyNo issues detectedNone
WarningCorrectable errors above thresholdInvestigate, schedule maintenance
FailureUncorrectable errors or hardware faultDrain workload, replace GPU

4. Diagnostics (dcgmi diag)

DCGM diagnostics run active tests against GPUs, ranging from quick checks to full stress tests:

# Diagnostic run levels:
# Level 1 — Quick (seconds): basic configuration check
dcgmi diag -g 1 -r 1

# Level 2 — Medium (minutes): memory bandwidth, PCIe, NVLink tests
dcgmi diag -g 1 -r 2

# Level 3 — Extended (hours): full stress test, burn-in level
dcgmi diag -g 1 -r 3

# Run specific tests only
dcgmi diag -g 1 -r 2 --plugin memtest --plugin pcie

# Output to file
dcgmi diag -g 1 -r 2 --debugLogFile /tmp/dcgm_diag.log
Diagnostic LevelDurationTests IncludedUse Case
Level 1<30 secondsConfig check, driver initQuick sanity after install
Level 22-10 minutesMemory, PCIe bandwidth, NVLink, ECCPost-maintenance verification
Level 3HoursFull compute stress, power, thermal, memoryPre-production burn-in
Exam: Which Level for Which Scenario?
Quick install verification = Level 1. After replacing a GPU = Level 2 or 3. Before accepting a new DGX into production cluster = Level 3 (extended burn-in).

5. Field Monitoring (dmon)

# Monitor all GPUs every 1000ms, fields: SM util, memory util, power
dcgmi dmon -e 203,204,155 -d 1000

# Monitor specific GPU group
dcgmi dmon -g 1 -e 203,204,155 -d 500

# Monitor and write to CSV file
dcgmi dmon -e 203,204,155 -d 1000 > /tmp/gpu_metrics.csv

# Watch NVLink throughput counters
dcgmi dmon -e 740,742 -d 1000   # NVLink RX/TX bandwidth per GPU

7. Policy Engine

DCGM policies automatically trigger actions (script, notification) when GPU conditions are met:

# View current policies on a group
dcgmi policy -g 1 -l

# Set policy: action when double-bit ECC error occurs
dcgmi policy -g 1 --set 0,0     # Parameters: ECC action, XID action

# Register a callback script
dcgmi policy -g 1 --reg          # Register action handler

# Policy conditions (configure thresholds):
# --ecc       : ECC memory error threshold
# --pci       : PCIe error threshold
# --maxpages  : GPU page retirement threshold
# --thermal   : Temperature threshold
# --power     : Power violation threshold

8. Exam Prep & Quick Reference

Must-Know for Exam (33% domain weight)
  • DCGM daemon: nv-hostengine or systemctl start nvidia-dcgm
  • Check all GPUs healthy: dcgmi health -g 1 -e
  • Quick verification after driver install: dcgmi diag -r 1
  • Post-GPU replacement verification: dcgmi diag -r 2 or -r 3
  • Monitor SM util in real-time: dcgmi dmon -e 203
  • ECC uncorrected errors field ID: 311
  • DCGM group with all GPUs: group ID = 1 (default)
  • DCGM integrates with Prometheus via DCGM Exporter (Kubernetes sidecar)
