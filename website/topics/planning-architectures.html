Planning & Reflection Architectures

Moving beyond reactive loops. Discover how agents Plan-and-Execute complex tasks upfront and use Self-Reflection (CRITIC) to iteratively improve their outputs.

1Plan-and-Execute Architecture

While ReAct is excellent for simple, step-by-step tasks, it suffers from "short-sightedness". A ReAct agent doesn't establish a long-term plan; it just decides what to do next.

Plan-and-Execute separates planning from execution:

  1. Planner Agent: Receives the user prompt and breaks it down into a multi-step execution plan.
  2. Executor Agent(s): Receives step 1, executes it using tools, returns the result. Then receives step 2, etc.
  3. Re-Planner (Optional): Reviews the trajectory. If a step failed, it dynamically alters the remaining steps.

This is highly tested in scale-out scenarios where execution steps can be run in parallel by different specific workers.

2Self-Reflection (Reflexion)

Agents naturally hallucinate or fail tool invocations. Self-reflection introduces a "thinking about thinking" loop. Before returning a final answer to the user, the agent passes its draft to a Reflector Node.

The Reflector acts as a critic: "Did I fully answer the user's prompt? Did the output format correctly? Are there hallucinations?" If it spots an error, it returns feedback to the generator to try again, rather than failing externally.

3CRITIC Framework

CRITIC (Validating LLMs using External Tools) is an advanced reflection pattern. Instead of using internal prompt-based reflection (which suffers from the same hallucination bias), CRITIC forces the agent to use tools (like Python execution or searching Wikipedia) to explicitly verify its own claims before outputting them.

4Exam Relevance

Exam Trap: You will be given a scenario where an agent needs to perform 5 independent API calls to gather data before summarizing it.

Question: Which architecture is best?

Answer: Plan-and-Execute. ReAct would do them sequentially and likely lose context. Plan-and-Execute generates the 5 tasks upfront, which can then be parallelized by executors.
