Agent Observability

Monitoring production agents in real-time — LangSmith distributed tracing, OpenTelemetry for AutoGen, Prometheus/Grafana for Triton metrics, performance drift detection, and NeMo Evaluator CI/CD integration.

1Why Observability Matters

An agent in production is not static. User query distribution changes, external tool APIs change, LLM behavior drifts after model updates, and context quality degrades. Without observability, you cannot detect these issues until users complain.

Observability answers: What decisions did the agent make? Why did it fail? How is latency trending? Which tool calls are failing most often? What is the cost per query this week vs last week?

2LangSmith — Agent Tracing

LangSmith is LangChain's observability platform. Every LLM call, tool invocation, retrieval step, and intermediate state is recorded as a "run" with a trace_id, run_id, latency, token count, cost estimate, and input/output payloads.

Python — Enable LangSmith Tracing

import os
os.environ["LANGCHAIN_TRACING_V2"] = "true"
os.environ["LANGCHAIN_API_KEY"] = "ls__xxxxx"
os.environ["LANGCHAIN_PROJECT"] = "ncp-aai-agent-v2"

# Now all LangChain/LangGraph calls are automatically traced
# → View at smith.langchain.com/projects
from langchain_nvidia_ai_endpoints import ChatNVIDIA
from langgraph.graph import StateGraph

# Every LLM call, tool call, and node execution is visible in LangSmith
llm = ChatNVIDIA(model="meta/llama-3.1-70b-instruct")
response = llm.invoke("What is paged KV cache?")

# LangSmith records:
# - trace_id, parent_run_id
# - input tokens: 12, output tokens: 87
# - latency: 1.23s
# - estimated cost: $0.000087
# - full input prompt + output response

4Triton + Prometheus Metrics

Triton Inference Server exposes Prometheus metrics on port 8002. These integrate with Grafana for real-time infrastructure dashboards:

Shell — Key Triton Prometheus Metrics

# Scrape with: curl http://triton-server:8002/metrics
nv_inference_request_success   # total successful inference requests
nv_inference_request_failure   # failed requests - alert on spike
nv_inference_queue_duration_us # time requests wait in queue
nv_inference_compute_infer_duration_us  # actual GPU compute time
nv_gpu_utilization             # GPU utilization % (0-100)
nv_gpu_memory_used_bytes       # GPU memory consumption
5Performance Drift Detection

Performance drift occurs when an agent's quality metrics change over time without any intentional code change. Common causes: change in user query distribution, LLM provider model updates, external tool API changes, knowledge base becoming stale.

Detection strategy: Run automated RAGAS evaluation on a fixed "golden dataset" (representative queries with known good answers) on a daily/weekly schedule via NeMo Evaluator CI/CD integration. Alert when any metric drops by more than 5% from baseline.

6OpenTelemetry for Agent Tracing

OpenTelemetry (OTel) is the vendor-neutral CNCF standard for distributed tracing. For agents that use multiple frameworks (LangGraph + custom tools + external APIs), OTel provides a unified tracing layer:

Python — OpenTelemetry Agent Instrumentation
from opentelemetry import trace
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor
from opentelemetry.exporter.otlp.proto.grpc.trace_exporter import OTLPSpanExporter

# Set up OTel tracer (sends to Jaeger/Grafana Tempo/Honeycomb)
provider = TracerProvider()
exporter = OTLPSpanExporter(endpoint="http://jaeger:4317")
provider.add_span_processor(BatchSpanProcessor(exporter))
trace.set_tracer_provider(provider)

tracer = trace.get_tracer("ncp-aai-agent")

# Instrument every agent step with a span
def reasoning_node(state):
    with tracer.start_as_current_span("reasoning_node") as span:
        span.set_attribute("agent.step", "reasoning")
        span.set_attribute("agent.message_count", len(state["messages"]))

        response = llm.invoke(state["messages"])

        span.set_attribute("agent.output_tokens", response.usage.completion_tokens)
        span.set_attribute("agent.model", "meta/llama-3.1-70b-instruct")
        return {"messages": [response]}

# Each span becomes visible in Jaeger/Grafana as a timeline
# Full trace shows: total latency, bottleneck nodes, error rates

Debugging Failed Agent Runs

Python — Structured Error Logging
import logging
import json
from datetime import datetime

logger = logging.getLogger("agent.production")

def log_agent_event(event_type: str, trace_id: str, **data):
    """Structured log for agent observability."""
    log_entry = {
        "timestamp": datetime.utcnow().isoformat(),
        "event_type": event_type,   # "tool_call", "llm_call", "error", "completion"
        "trace_id": trace_id,
        **data
    }
    logger.info(json.dumps(log_entry))

# Usage:
log_agent_event("tool_call", trace_id=trace_id,
    tool_name="web_search", args={"query": "..."}, status="success",
    duration_ms=230, result_size_chars=4521)

log_agent_event("error", trace_id=trace_id,
    error_type="ToolTimeoutError", tool_name="code_executor",
    duration_ms=10000, retry_attempt=2)
