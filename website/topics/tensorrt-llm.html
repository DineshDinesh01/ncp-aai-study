TensorRT-LLM

NVIDIA's open-source library for LLM inference optimization — quantization (INT4/INT8/FP8), in-flight batching, paged KV cache, and speculative decoding. The engine behind NVIDIA NIMs and Triton's LLM backend.

1What Is TensorRT-LLM?

TensorRT-LLM is NVIDIA's open-source Python library that compiles large language model inference into highly optimized TensorRT engines. It applies quantization, kernel fusion, attention optimization, and memory management techniques to maximize throughput and minimize latency on NVIDIA GPUs.

Key output: a compiled .engine file that can be served via Triton Inference Server or NVIDIA NIMs.

🎯 Exam Key Fact

FP8 quantization is only available on H100 and Ada Lovelace GPUs. It has lower accuracy impact than INT8 KV cache and is the recommended approach for Hopper/Ada deployments.

3In-Flight Batching & Paged KV Cache

In-flight batching (also called continuous batching or iteration-level batching): instead of waiting for all requests in a batch to finish before starting new ones, TensorRT-LLM dynamically adds new requests and evicts completed ones at every decode step. This maximizes GPU utilization and dramatically reduces P50/P99 latency at high load.

Paged KV cache: inspired by OS virtual memory paging. The KV cache is split into fixed-size "pages" (blocks) that can be allocated non-contiguously in GPU memory. Benefits: near-zero KV cache waste, efficient sharing of KV pages across requests with the same prefix (prefix caching), and support for much longer sequences.

Python — TensorRT-LLM Build Config

# trtllm-build command
trtllm-build \
  --checkpoint_dir ./llama3-8b-hf \
  --output_dir ./engine \
  --dtype bfloat16 \
  --use_fused_mlp \
  --use_paged_context_fmha enable \     # paged KV cache
  --max_batch_size 64 \
  --max_input_len 4096 \
  --max_output_len 512 \
  --quantization fp8                      # FP8 on H100
4Speculative Decoding

Speculative decoding uses a small, fast "draft model" to predict multiple future tokens, which are then verified in parallel by the large target model in a single forward pass. If draft tokens are accepted, multiple tokens are generated per LLM call — reducing latency at low batch sizes by 2–4x.

TensorRT-LLM supported algorithms: Draft/Target models, EAGLE (Enhanced Accurate Generative Language Engine), NGram (pattern matching from context), MTP (Multi-Token Prediction).

6Common Mistakes & Exam Angles
❌ Using FP8 on Non-H100 GPU

FP8 is only supported on H100 and Ada GPUs. On A100 or earlier, use INT8 (SmoothQuant) or W4A16 (AWQ).

🎯 Exam Key Points
  • AWQ = Activation-aware Weight Quantization = W4A16
  • SmoothQuant = W8A8 (both weights and activations are INT8)
  • In-flight batching = continuous batching = iteration-level batching (same concept)
  • Paged KV cache: near-zero memory waste, enables prefix caching
  • Speculative decoding reduces latency at low batch sizes (not throughput-optimal)
  • LoRA adapters must be merged before TRT-LLM compilation
