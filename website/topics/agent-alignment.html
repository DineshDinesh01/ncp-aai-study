Agent Alignment (RLHF & DPO)

Fine-tuning teaches an LLM a new format. Alignment teaches an agent what humans actually prefer it to do.

Table of Contents
  • 1 SFT vs Alignment
  • 2 RLHF Framework
  • 3 Direct Preference Optimization (DPO)
  • 4 NeMo Customizer alignment

1SFT vs Alignment

Supervised Fine-Tuning (SFT) teaches a model how to act (e.g. outputting valid JSON). But it does not teach a model whether its actions are safe, polite, or helpful. Alignment is the secondary training phase that aligns the LLM's vast knowledge with human safety values and behavioral preferences.

2RLHF Framework

Reinforcement Learning from Human Feedback (RLHF) requires training a separate "Reward Model" based on human raters grading two LLM outputs. The Base LLM then generates responses, the Reward Model scores them, and PPO (Proximal Policy Optimization) updates the LLM to maximize the reward score.

RLHF is complex because it requires managing multiple models simultaneously in memory.

3Direct Preference Optimization (DPO)

DPO mathematically bypasses the Reward Model entirely. By formatting data as (Prompt, Chosen_Response, Rejected_Response), DPO treats the language model itself as the reward model, directly updating its weights to prefer the 'Chosen' path.

DPO is vastly more memory-efficient and stable to implement than RLHF, making it the industry standard for modern Agent alignment pipelines.

4Exam Question Targets

Exam Trap: You need to align a newly tuned customer support Agent to prefer polite refusals over hallucinatory promises. You have a dataset of 10k "Good Answer vs Bad Answer" pairs. Which method uses less memory?

Answer: DPO. DPO does not require training a distinct Reward network and runs PPO asynchronously, saving massive amounts of VRAM over standard RLHF.
