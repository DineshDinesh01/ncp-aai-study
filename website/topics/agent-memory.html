Agentic Memory & Checkpointing

Stateless LLMs forget everything between calls. Learn how memory frameworks transform chatbots into personalized assistants by persisting state across execution threads.

1Working Memory (The Prompt)

An LLM's true memory is its physical context window. Any state outside the prompt is lost. As tasks become complex, the agent builds a "working memory" directly inside the prompt (often via a scratchpad).

However, you cannot infinitely cram data into working memory because of token limits and performance degradation (the "lost in the middle" phenomenon). You must externalize memory.

2Episodic Memory (Conversation History)

Episodic memory is raw, sequential history. You remember the exact sequence of events ("I said X, then you said Y").

This is commonly managed via rolling context buffers (e.g. ConversationBufferWindowMemory in LangChain or standard array trimming). To prevent overflow, older messages are either truncated or summarized recursively into a single "past interactions" summary note.

3Semantic Memory (Vector Stores)

Semantic memory extracts meaning from episodes and saves them as factual knowledge. Rather than memorizing the exact words "My name is John and I live in Seattle" from 4 days ago, a background job extracts the facts "User Name: John", "User Location: Seattle".

These facts are then embedded and stored in a vector database. During runtime, the agent performs a similarity search on the incoming prompt against the vector database to inject relevant semantic facts back into the working memory.

4State Checkpointing (LangGraph)

While semantic memory stores facts, a Checkpoint stores the exact computational state of an agentic workflow. In frameworks like LangGraph, you construct a state dictionary. At every node execution step, LangGraph saves the exact values of this tree into a persistent Postgres or SQLite DB.

pythonLangGraph Saver
from langgraph.checkpoint.sqlite import SqliteSaver

memory = SqliteSaver.from_conn_string(":memory:")
app = builder.compile(checkpointer=memory)

# Resume earlier thread execution
app.invoke({"messages": [new_input]}, config={"configurable": {"thread_id": "user_123"}})

5Exam Question Targets

Exam Trap: Which memory type allows an agent to "resume" a paused multi-step internal process that was halted awaiting user approval?

Answer: State Checkpointing. Episodic/Semantic memory only stores facts, not the graph execution frame. State Checkpointers save the exact graph node traversal ID.
