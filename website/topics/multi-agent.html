Multi-Agent Orchestration

Designing systems where multiple specialized agents collaborate — AutoGen Mixture-of-Agents, CrewAI Flows, supervisor pattern, semantic routing, hierarchical vs peer-to-peer vs blackboard coordination.

1What Is Multi-Agent Orchestration?

A multi-agent system is an architecture where multiple distinct AI agents collaborate to solve tasks no single agent could handle well. Each agent has a specialized role, model, or capability. An orchestrator coordinates their work.

Key patterns:

PatternStructureBest For
HierarchicalManager agent → worker agentsTask decomposition, delegation
Peer-to-PeerAgents communicate directly via shared stateCollaborative reasoning, critique
BlackboardShared knowledge board all agents read/writeComplex, multi-phase problem solving
Mixture-of-Agents (MoA)Multiple LLMs → aggregatorConsensus, ensemble quality improvement
2AutoGen — Conversational Multi-Agent
Python — AutoGen 3-Agent Setup

import autogen

config = {"model": "meta/llama-3.1-70b-instruct",
          "base_url": "https://integrate.api.nvidia.com/v1",
          "api_key": "nvapi-xxx"}

# Orchestrator (human proxy)
user_proxy = autogen.UserProxyAgent(
    name="User",
    human_input_mode="NEVER",      # fully automated
    code_execution_config={"work_dir": "coding"}
)

# Specialized agents
researcher = autogen.AssistantAgent(
    name="Researcher",
    system_message="Research facts and provide accurate information.",
    llm_config={"config_list": [config]}
)

critic = autogen.AssistantAgent(
    name="Critic",
    system_message="Review responses and identify errors or gaps. Be concise.",
    llm_config={"config_list": [config]}
)

# GroupChat: all agents see all messages
groupchat = autogen.GroupChat(
    agents=[user_proxy, researcher, critic],
    messages=[],
    max_round=6
)
manager = autogen.GroupChatManager(groupchat=groupchat,
                                    llm_config={"config_list": [config]})

# Start conversation
user_proxy.initiate_chat(manager, message="Explain HNSW indexing.")
3CrewAI — Role-Based Production Agents

CrewAI uses an Agents → Tasks → Crew architecture. Each Agent has a role, goal, and backstory. The Crew executes Tasks in sequence or in parallel. CrewAI Flows provide explicit state management and conditional branching for production workflows.

💡 CrewAI Production Scale

CrewAI has executed 10 million+ agents in 30 days in real enterprise deployments. It's production-ready, not just a demo framework. Used by major enterprises for customer service, research, and code review pipelines.

4Semantic Router — Intelligent Agent Dispatch

A semantic router routes incoming user requests to the most appropriate agent based on semantic similarity rather than keyword matching. Each agent has a description; the router embeds the user request and finds the closest agent description.

Python — Semantic Router Pattern

from sentence_transformers import SentenceTransformer
import numpy as np

model = SentenceTransformer("all-MiniLM-L6-v2")

# Agent registry
agents = {
    "financial_agent": "Handle financial analysis, stock prices, portfolio management",
    "code_agent": "Write, review, and debug Python, JavaScript, SQL code",
    "research_agent": "Search and summarize academic papers and technical documents",
}

def route_to_agent(user_query: str) -> str:
    query_emb = model.encode(user_query)
    agent_embs = {name: model.encode(desc) for name, desc in agents.items()}

    scores = {
        name: np.dot(query_emb, emb) / (np.linalg.norm(query_emb) * np.linalg.norm(emb))
        for name, emb in agent_embs.items()
    }
    return max(scores, key=scores.get)

print(route_to_agent("What is the P/E ratio of Apple stock?"))
# → financial_agent
print(route_to_agent("Write a FastAPI endpoint for user authentication"))
# → code_agent
5Exam Angles
🎯 Exam Key Points — Agent Architecture (15%)
  • AutoGen: UserProxyAgent, AssistantAgent, GroupChatManager — conversational, event-driven
  • CrewAI: Agent → Task → Crew — role-based, structured workflows
  • AutoGen consolidated into Microsoft Agent Framework (Oct 2025) — enters maintenance mode
  • Hierarchical pattern: manager agent delegates to workers (CrewAI's process=hierarchical)
  • Blackboard pattern: shared state all agents read/write — common in complex planning
  • Semantic router: embedding-based routing vs keyword-based routing
  • Stateless subagents: recommended for parallel execution and predictable behavior
