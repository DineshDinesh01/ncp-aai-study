Agent Evaluation & RAGAS

How to measure whether your agent is actually working — RAGAS metrics for RAG eval, agent-specific KPIs (task completion rate, reasoning accuracy, latency per step, cost per interaction), LLM-as-a-judge, A/B testing, and NVIDIA NeMo Evaluator.

1Why Evaluation Matters

Without rigorous evaluation, you cannot tell if your agent is improving or degrading after changes. The NCP-AAI exam treats evaluation as a first-class skill — 13% of the exam covers benchmarking, A/B testing, and continuous monitoring. In 2025, 70% of engineers using RAG in production integrate evaluation into their CI/CD pipelines.

2RAGAS — RAG System Evaluation

RAGAS (Retrieval-Augmented Generation Assessment) is the standard open-source framework for evaluating RAG pipelines. It is "reference-free" — it uses LLMs to evaluate quality without requiring hand-labeled ground truth for every question.

Python — RAGAS Evaluation

from ragas import evaluate
from ragas.metrics import (
    context_precision, context_recall,
    faithfulness, answer_relevancy
)
from datasets import Dataset

data = {
    "question": ["What port does Triton gRPC use?"],
    "answer": ["Triton uses gRPC on port 8001."],
    "contexts": [["NVIDIA Triton uses gRPC on port 8001 (preferred for production) and HTTP on port 8000."]],
    "ground_truth": ["Triton gRPC port is 8001."]
}
dataset = Dataset.from_dict(data)

results = evaluate(
    dataset,
    metrics=[context_precision, context_recall, faithfulness, answer_relevancy]
)
print(results)
# {'context_precision': 1.0, 'faithfulness': 1.0, 'answer_relevancy': 0.97, ...}

4LLM-as-a-Judge

Using an LLM to evaluate another LLM's output is becoming the standard approach in 2025 (used by 70%+ of production RAG systems). The judge LLM scores the response on criteria like accuracy, relevance, coherence, and safety on a 1–5 scale.

Python — LLM-as-a-Judge Pattern

from openai import OpenAI

client = OpenAI(base_url="https://integrate.api.nvidia.com/v1", api_key="nvapi-xxx")

def llm_judge(question: str, answer: str, context: str) -> dict:
    prompt = f"""Rate this RAG answer on a scale of 1-5 for each criterion.

Question: {question}
Retrieved Context: {context}
Agent Answer: {answer}

Criteria:
1. Faithfulness (is the answer supported by the context?): 1-5
2. Relevancy (does it answer the question?): 1-5
3. Completeness (is all important info included?): 1-5

Respond ONLY as JSON: {{"faithfulness": N, "relevancy": N, "completeness": N, "reasoning": "..."}}"""

    response = client.chat.completions.create(
        model="meta/llama-3.1-70b-instruct",
        messages=[{"role": "user", "content": prompt}],
        response_format={"type": "json_object"}
    )
    return response.choices[0].message.content

6A/B Testing Agent Strategies

A/B testing agent strategies without impacting production means running two agent variants on shadow traffic or a small percentage of real traffic, comparing their metric distributions statistically before promoting the winner.

🎯 Exam Key Points — Evaluation & Tuning (13%)
  • RAGAS Faithfulness: is answer grounded in retrieved context? (detects hallucination)
  • RAGAS Context Precision: signal-to-noise in retrieved chunks
  • RAGAS Context Recall: was all necessary info retrieved?
  • LLM-as-a-judge: standard in 2025, scales eval without human annotation
  • Agent KPIs: task completion rate, reasoning accuracy, latency per step, cost per interaction
  • NeMo Evaluator: 100+ benchmarks + LLM-as-a-judge + CI/CD integration
  • A/B test on shadow traffic to compare agent variants without production impact
