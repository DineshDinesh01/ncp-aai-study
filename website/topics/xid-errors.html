XID Error Reference

NVIDIA GPU XID error codes — hardware fault indicators logged in dmesg, nvidia-smi, and DCGM. Know the critical codes, their causes, and the right remediation action for each.

1. What Are XID Errors?

XID (eXternal Interrupt Description) errors are hardware and software fault codes logged by the NVIDIA GPU driver when it detects an abnormal condition on the GPU. They appear in the kernel ring buffer (dmesg), system logs, and DCGM.

XID Errors Are Not All Equal
Some XIDs are transient and expected under normal operation. Others (especially XID 48 double-bit ECC, XID 79 GPU off bus) indicate hardware failure requiring GPU replacement. Knowing which is which is critical for the exam.

2. Detection Commands

# Primary: check dmesg for XID errors
sudo dmesg | grep -i "NVRM: Xid"
sudo dmesg | grep "Xid"

# Example XID error in dmesg:
# NVRM: Xid (PCI:0000:03:00): 48, pid='', name=, Ch 00000008, ...
# NVRM: Xid (PCI:0000:03:00): 79, pid=0, name=, ...

# Real-time monitoring
sudo dmesg -w | grep Xid

# Check nvidia-smi for ECC error counts
nvidia-smi --query-gpu=gpu_bus_id,ecc.errors.corrected.aggregate.total,ecc.errors.uncorrected.aggregate.total --format=csv

# DCGM — check for XID events
dcgmi health -g 1 -e

# Check system journal for NVRM XID entries
journalctl -k | grep "Xid"
journalctl -k --since="1 hour ago" | grep -i "nvrm\|xid\|nvidia"

# Nvidia SMI event log
nvidia-smi events -i 0   # Watch events for GPU 0 (keep running)

4. ECC Errors (XID 48, XID 63)

ECC (Error Correcting Code) is built into HBM GPU memory. It classifies errors into two types:

Error TypeXIDBits FlippedCorrectable?Action
SBE (Single-Bit Error)63 (page retirement)1 bitYes — corrected by ECC hardwarePage is retired; monitor count
DBE (Double-Bit Error)482+ bitsNo — data is corruptWorkload killed; GPU may need replacement
 
# Check ECC error counts
nvidia-smi -q -d ECC -i 0

# Key fields:
# Volatile ECC Errors:   errors since last driver load
# Aggregate ECC Errors:  total since GPU was manufactured

# Volatile counts reset on driver reload/reboot
# Aggregate counts are permanent hardware counters

# Page retirement status
nvidia-smi --query-remapped-rows=gpu_bus_id,remapped_rows.correctable,remapped_rows.uncorrectable --format=csv

# Rule of thumb for GPU replacement:
# > 0 DBE uncorrectable aggregate errors = investigate immediately
# Persistent SBE row retirements (growing) = schedule GPU replacement
# All HBM rows retired = GPU must be replaced
XID 48 = Critical — GPU May Need Replacement
A single XID 48 (double-bit ECC) means a CUDA workload was terminated with corrupted data. If the aggregate DBE count is non-zero and growing, plan for GPU replacement. NVIDIA support typically uses persistent DBE count to authorize RMA.

5. Memory Faults (XID 8, XID 31)

# XID 8: GPU memory access fault — usually application bug
# Look for stack trace in dmesg after the XID
sudo dmesg | grep -A 5 "Xid.*8,"

# XID 31: GPU memory page fault — unified memory issue
# Check if application uses CUDA Unified Memory
grep -r "cudaMallocManaged\|torch.zeros(" app_code/

# Test GPU memory integrity (use dcgmi diag -r 3 or cuda memtest)
dcgmi diag -g 1 -r 3 --plugin memtest

# nvidia-smi memory error check
nvidia-smi --query-gpu=index,memory.used,ecc.errors.corrected.volatile.total --format=csv

6. Engine Errors (XID 13, XID 32)

# XID 13: Graphics/compute engine exception
# Usually means GPU compute unit encountered unexpected state

# Check if it's a driver issue
nvidia-smi --query-gpu=driver_version --format=csv
# Compare against latest stable driver for your GPU generation

# XID 32: Invalid push buffer — driver command stream corrupted
# This often indicates driver bugs — check release notes for fixes

# Check for recurring XID 13 on specific GPU
sudo dmesg | grep "Xid.*13" | awk '{print $4}' | sort | uniq -c

7. Bus & Driver Errors (XID 79, XID 119)

# XID 79: GPU fallen off PCIe bus — catastrophic failure
# The GPU is no longer visible; nvidia-smi will show fewer GPUs

# Detection
nvidia-smi -L          # Count GPUs — if fewer than 8, one fell off bus
sudo dmesg | grep "Xid.*79"

# Remediation steps for XID 79:
# 1. Power off system
# 2. Reseat the GPU in PCIe slot
# 3. Check PCIe power connectors
# 4. Power on and verify GPU returns to nvidia-smi -L
# 5. If recurring → hardware failure, replace GPU

# XID 119: GSP (GPU System Processor) RPC timeout
# GSP is embedded ARM processor running GPU firmware

# Check driver vs firmware version
nvidia-smi --query-gpu=driver_version --format=csv
cat /proc/driver/nvidia/version

# Update driver and firmware together
apt install nvidia-driver-550     # Example for Ubuntu
# Firmware updates may also be needed via firmware package

8. Triage Flowchart

XID Triage Decision Tree
  1. Check dmesg | grep Xid → note XID number and GPU PCI address
  2. Is the GPU still visible? (nvidia-smi -L)
    • No → XID 79 (GPU off bus) → power off, reseat, replace if recurring
    • Yes → continue
  3. Check ECC error counts (nvidia-smi -q -d ECC)
    • DBE aggregate > 0 → XID 48 → plan GPU replacement
    • SBE growing rapidly → XID 63 → monitor, schedule replacement
  4. Check Fabric Manager (systemctl status nvidia-fabricmanager)
    • Failed → XID 74 (NVLink) → check FM version, restart FM
  5. Driver/firmware → XID 119 → verify version match, update
  6. Application fault (XID 8, 13, 31) → debug workload, update driver

9. Thermal Troubleshooting

# Monitor GPU temperatures
nvidia-smi --query-gpu=index,temperature.gpu,temperature.memory --format=csv
nvidia-smi dmon -s t    # Continuous temperature monitoring

# Temperature thresholds for A100/H100:
# Normal operation:    40-80°C GPU core
# Warning threshold:   83°C (may start throttling)
# Critical/shutdown:   90°C (thermal emergency, GPU shuts down)
# HBM memory limit:    95°C

# Check for thermal throttling
nvidia-smi --query-gpu=clocks_throttle_reasons.hw_thermal_slowdown --format=csv
# "Active" = GPU is throttling due to heat

# Check power limit and current draw
nvidia-smi --query-gpu=power.limit,power.draw --format=csv

# Fan status (for systems with fans, not liquid cooled)
ipmitool sdr type Fan    # Check fan speeds via BMC

# Thermal event in SEL log
ipmitool sel list | grep -i "thermal\|temp"

10. Exam Prep & Quick Reference
