AutoGen

Conversational multi-agent framework — UserProxyAgent, AssistantAgent, GroupChatManager, termination conditions, and human-in-the-loop patterns with NVIDIA NIMs.

1What Is AutoGen?

AutoGen is a Microsoft Research framework for building conversational multi-agent systems. Unlike CrewAI (task-centric) or LangGraph (graph-centric), AutoGen models agent collaboration as a structured conversation — agents take turns sending messages to each other, with tool execution and code running embedded in the conversation flow.

💡 2025 Status Update

AutoGen was consolidated into the Microsoft Agent Framework (AG2) in late 2025. The original AutoGen package enters maintenance mode. The concepts and APIs from AutoGen 0.2/0.3 remain heavily tested on the NCP-AAI exam, so study them thoroughly.

AutoGen's core insight: most multi-agent collaboration can be modeled as agents chatting — one writes code, another executes it, a third reviews the result, and so on. This maps naturally to how human teams communicate.

2Why AutoGen Exists

AutoGen was designed to solve problems that single-agent loops handle poorly:

ProblemAutoGen Solution
Code needs to be executed and results fed backUserProxyAgent runs code and returns stdout/stderr to the conversation
Multiple expertise areas needed simultaneouslyGroupChat with specialist AssistantAgents
Human approval needed mid-executionhuman_input_mode="ALWAYS" pauses for user input
Different LLMs for different subtasksEach agent can have its own llm_config
Self-correction on errorsAgents see full conversation history and can revise based on error messages
3Core Agent Types

ConversableAgent — The Base Class

All AutoGen agents inherit from ConversableAgent. It handles message passing, termination detection, and conversation history management.

UserProxyAgent

Represents the human or automated executor side. Its primary role is to execute code produced by assistant agents and return results.

Python — UserProxyAgent Modes
import autogen

# human_input_mode controls when humans are asked to intervene
user_proxy = autogen.UserProxyAgent(
    name="User",
    human_input_mode="NEVER",       # Fully automated — never asks human
    # human_input_mode="ALWAYS",    # Ask human on every turn
    # human_input_mode="TERMINATE", # Only ask when agent wants to stop

    # Code execution configuration
    code_execution_config={
        "work_dir": "coding",        # Directory where code runs
        "use_docker": False,         # Set True for sandboxed execution
        "timeout": 60,               # Max seconds per code execution
        "last_n_messages": 3,        # Only look at last N messages for code blocks
    },

    # Termination conditions
    is_termination_msg=lambda msg: (
        msg.get("content", "").rstrip().endswith("TERMINATE")
    ),
    max_consecutive_auto_reply=10,   # Stop after N auto-replies without human
)

AssistantAgent

The LLM-backed agent that reasons, plans, writes code, and generates responses. Each AssistantAgent has its own llm_config and system_message.

Python — AssistantAgent Definition
assistant = autogen.AssistantAgent(
    name="DataScientist",
    system_message="""You are an expert data scientist specializing in LLM evaluation.
    When writing code, always include error handling.
    End your final response with TERMINATE to signal completion.""",
    llm_config={
        "config_list": [{
            "model": "meta/llama-3.1-70b-instruct",
            "base_url": "https://integrate.api.nvidia.com/v1",
            "api_key": "nvapi-xxx",
        }],
        "temperature": 0.1,
        "timeout": 120,
        "cache_seed": 42,            # Deterministic caching for reproducibility
    },
    max_consecutive_auto_reply=5,
)

4GroupChat and GroupChatManager

For more than 2 agents, AutoGen uses GroupChat to manage a shared conversation space, and GroupChatManager to decide which agent speaks next.

Python — GroupChat Configuration
groupchat = autogen.GroupChat(
    agents=[user_proxy, researcher, coder, reviewer],
    messages=[],
    max_round=12,                    # Maximum conversation turns

    # Speaker selection strategies:
    speaker_selection_method="auto", # LLM picks next speaker (default)
    # speaker_selection_method="round_robin"  # Agents take turns in order
    # speaker_selection_method="random"       # Random selection

    # Advanced: hint next speaker
    # Use when you want structured turn-taking but with some flexibility
    allow_repeat_speaker=False,      # Prevent same agent twice in a row
)

manager = autogen.GroupChatManager(
    groupchat=groupchat,
    llm_config={"config_list": [nvidia_config]},
    # Manager uses its LLM to select the next most-appropriate speaker
)

5Full Python Example — Code Review Pipeline
Python — AutoGen Code Review Multi-Agent System
import autogen

# NVIDIA NIM configuration
nvidia_config = {
    "model": "meta/llama-3.1-70b-instruct",
    "base_url": "https://integrate.api.nvidia.com/v1",
    "api_key": "nvapi-xxx",
    "temperature": 0.1,
}

# Agent 1: User Proxy (executes code, represents human)
user_proxy = autogen.UserProxyAgent(
    name="UserProxy",
    human_input_mode="NEVER",
    code_execution_config={"work_dir": "workspace", "use_docker": False},
    is_termination_msg=lambda x: x.get("content", "").endswith("TERMINATE"),
    max_consecutive_auto_reply=8,
)

# Agent 2: Software Engineer (writes code)
engineer = autogen.AssistantAgent(
    name="Engineer",
    system_message="""You are a senior Python engineer. Write clean, efficient code.
    Always include docstrings and type hints. Test your logic before finalizing.
    End with TERMINATE only when the final, reviewed code is complete.""",
    llm_config={"config_list": [nvidia_config]},
)

# Agent 3: Code Reviewer (checks for bugs and style)
reviewer = autogen.AssistantAgent(
    name="Reviewer",
    system_message="""You are a meticulous code reviewer. Check for:
    1. Logic errors and edge cases
    2. Security vulnerabilities
    3. Performance issues
    4. Missing error handling
    Provide specific, actionable feedback. Do not approve code with critical issues.""",
    llm_config={"config_list": [nvidia_config]},
)

# Agent 4: QA Engineer (writes tests)
qa_engineer = autogen.AssistantAgent(
    name="QA_Engineer",
    system_message="""You write comprehensive pytest test cases. Cover:
    - Happy path scenarios
    - Edge cases (empty input, large input, invalid types)
    - Error conditions
    Always verify tests pass before signaling completion.""",
    llm_config={"config_list": [nvidia_config]},
)

# GroupChat: all agents collaborate
groupchat = autogen.GroupChat(
    agents=[user_proxy, engineer, reviewer, qa_engineer],
    messages=[],
    max_round=15,
    speaker_selection_method="auto",  # Manager LLM picks next speaker
)

manager = autogen.GroupChatManager(
    groupchat=groupchat,
    llm_config={"config_list": [nvidia_config]},
)

# Start the pipeline
user_proxy.initiate_chat(
    manager,
    message="""Write a Python function that:
    1. Takes a list of LLM API calls with token counts
    2. Calculates total cost using a per-token pricing model
    3. Returns a summary dict with total_tokens, total_cost, avg_cost_per_call

    After Engineer writes it, Reviewer checks it, then QA_Engineer writes tests.
    End with TERMINATE when all three are complete."""
)

# Access full conversation history
for msg in groupchat.messages:
    print(f"[{msg['name']}]: {msg['content'][:100]}...")
6NVIDIA NIM Integration

AutoGen uses llm_config with a config_list format. NVIDIA NIMs are OpenAI-compatible, so they slot in directly:

Python — Multiple NVIDIA Models per Agent
# Use different NVIDIA models for different agent roles
# AutoGen tries configs in order — first that succeeds is used

engineer_config = {
    "config_list": [
        {   # Primary: powerful model for code writing
            "model": "meta/llama-3.1-70b-instruct",
            "base_url": "https://integrate.api.nvidia.com/v1",
            "api_key": "nvapi-xxx",
        },
        {   # Fallback: smaller, faster model
            "model": "meta/llama-3.1-8b-instruct",
            "base_url": "https://integrate.api.nvidia.com/v1",
            "api_key": "nvapi-xxx",
        },
    ],
    "temperature": 0.0,   # Deterministic for code generation
    "cache_seed": 42,     # Cache LLM responses for cost savings in dev
}

reviewer_config = {
    "config_list": [{
        # NVIDIA-tuned for critique/RLHF — excellent for review tasks
        "model": "nvidia/llama-3.1-nemotron-70b-instruct",
        "base_url": "https://integrate.api.nvidia.com/v1",
        "api_key": "nvapi-xxx",
    }],
    "temperature": 0.1,
}

# Disable caching in production
# (cache_seed=None ensures fresh LLM calls)
production_config = {"config_list": [...], "cache_seed": None}
7Lab Exercise
🔬 Lab: AutoGen Code Generation + Review + Test Pipeline

Objective: Build a 3-agent AutoGen system that writes, reviews, and tests Python code.

  1. Install: pip install pyautogen
  2. Set up llm_config pointing to NVIDIA NIMs
  3. Create Engineer + Reviewer + UserProxy (with code execution enabled)
  4. Run a 2-agent chat: UserProxy asks Engineer to write a RAGAS evaluation script
  5. Observe how Engineer's code is executed by UserProxy and errors fed back
  6. Extension: Add a 4th QA agent; use GroupChat with speaker_selection_method="auto"
  7. Check groupchat.messages — trace the full conversation to understand turn-taking

Key observation: Watch how UserProxy sends code execution errors back into the conversation, allowing Engineer to self-correct — this is AutoGen's core value proposition.

8Common Mistakes
MistakeImpactFix
No termination conditionConversation runs until max_round, wasting tokensAlways set is_termination_msg + TERMINATE in system_message
human_input_mode="ALWAYS" in productionPipeline blocks waiting for human input that never comesUse NEVER for automated; TERMINATE for supervised
use_docker=False in untrusted environmentsAgents can execute arbitrary code on hostUse Docker sandbox for any external/untrusted code execution
All agents same LLMExpensive; no specialization benefitUse larger models for complex reasoning, smaller for summaries
Forgetting cache_seed=None in productionProduction calls hit stale cacheSet cache_seed=None for live deployments
max_round too lowComplex tasks cut off before completionSet max_round based on expected task complexity (10-20 typical)
9Exam Prep
🎯 Exam Key Points — AutoGen (Agent Architecture 15%)
  • UserProxyAgent: represents human/executor side; can execute code via code_execution_config
  • AssistantAgent: LLM-backed; generates text, code, plans; does NOT execute code itself
  • GroupChatManager: uses its own LLM to select next speaker; manages GroupChat turn-taking
  • human_input_mode: NEVER (fully automated), ALWAYS (every turn), TERMINATE (only on stop signal)
  • Termination: TERMINATE keyword in message content; captured by is_termination_msg lambda
  • max_round: hard limit in GroupChat; max_consecutive_auto_reply: per-agent limit
  • Self-correction loop: Agent writes code → UserProxy executes → error returned as message → Agent revises
  • cache_seed: set integer for dev (deterministic, cached); None for production
  • 2025 status: AutoGen consolidated into Microsoft Agent Framework (AG2) — enters maintenance mode
  • vs CrewAI: AutoGen = conversational, code-execution-centric; CrewAI = task-assignment, role-centric
