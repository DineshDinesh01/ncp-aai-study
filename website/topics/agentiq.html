NVIDIA AgentIQ

Open-source toolkit for composable, observable agentic workflows — profiling, evaluation, langgraph_wrapper, and how AgentIQ fits alongside NIMs in the NVIDIA stack.

1What Is AgentIQ?

NVIDIA AgentIQ is an open-source Python toolkit for building, composing, profiling, and evaluating agentic AI workflows. It is framework-agnostic — it wraps existing LangGraph, CrewAI, or custom agents and adds NVIDIA-native capabilities on top.

Key capabilities:

  • Composable workflows — define agents as YAML-configurable functions, compose them into pipelines
  • Built-in profiling — wall-clock timing, token counts, and cost per agent step
  • Evaluation integration — connect to NeMo Evaluator for accuracy benchmarking
  • Framework wrapping — wrap existing LangGraph graphs via langgraph_wrapper
  • Concurrent execution — run multiple agent functions in parallel with async primitives
2AgentIQ vs NIMs — The Critical Distinction
🎯 This is a direct exam question

The NCP-AAI exam explicitly tests whether you understand the difference between AgentIQ and NIMs. Many candidates confuse them.

DimensionNVIDIA NIMsNVIDIA AgentIQ
What it isContainerized inference microservices for model servingOpen-source Python toolkit for agentic workflows
Primary jobServe LLM/model inference via OpenAI-compatible APIOrchestrate, profile, and evaluate agent pipelines
DeploymentDocker container on GPU infrastructurePython library in your application code
Input/OutputHTTP REST requests → model predictionsAgent function calls → structured workflow results
RelationshipAgentIQ workflows CALL NIM endpoints for LLM inference
💡 Analogy

NIMs = the engine (model serving). AgentIQ = the vehicle (workflow orchestration). You need both: AgentIQ orchestrates the agent logic; NIMs serve the LLM calls the agent makes.

3AgentIQ Architecture

AgentIQ is built around three layers:

Functions
Python callables with AgentIQ decorators
→
Workflows
Composed pipelines (YAML or code config)
→
Profiler + Evaluator
Automatic timing, cost, accuracy metrics
Python — AgentIQ Function Registration
from aiq.builder.function_info import FunctionInfo
from aiq.builder.framework_enum import LLMFrameworkEnum

# Register a Python function as an AgentIQ tool/agent function
@FunctionInfo.create(
    name="research_agent",
    description="Research a technical topic using web search",
    input_type=str,
    output_type=str,
)
async def research_agent(query: str, **kwargs) -> str:
    """AgentIQ wraps this function with profiling and evaluation."""
    llm = kwargs["llm"]
    tools = kwargs["tools"]
    # Your agent logic here
    result = await run_react_loop(llm, tools, query)
    return result
4Workflow Types

AgentIQ supports multiple workflow types for different use cases:

Workflow TypeDescriptionUse Case
react_agentStandard ReAct loop with registered toolsSingle-agent tool use
langgraph_wrapperWraps an existing LangGraph StateGraphAdding AgentIQ profiling to existing LangGraph agents
tool_calling_agentOpenAI-style function calling agentStructured tool-call workflows
customFully custom Python async functionArbitrary agentic logic
5Code Examples

Using langgraph_wrapper

Python — Wrap Existing LangGraph Agent with AgentIQ
from langgraph.graph import StateGraph, END
from aiq.builder.workflow_builder import WorkflowBuilder

# Your existing LangGraph agent (unchanged)
def build_my_graph():
    graph = StateGraph(AgentState)
    graph.add_node("researcher", research_node)
    graph.add_node("writer", write_node)
    graph.add_edge("researcher", "writer")
    graph.add_edge("writer", END)
    return graph.compile()

# Wrap it with AgentIQ for profiling + evaluation
builder = WorkflowBuilder()
builder.add_workflow(
    workflow_type="langgraph_wrapper",
    workflow_config={
        "graph_factory": build_my_graph,
        "input_key": "messages",
        "output_key": "final_answer",
    }
)

# AgentIQ now adds:
# - Step-level timing (wall-clock and CPU)
# - Token counts per node
# - Cost estimates per NIM call
# - Evaluation hooks for NeMo Evaluator
workflow = builder.build()
result = await workflow.run({"messages": [{"role": "user", "content": "Research LLM quantization"}]})

print(result.output)
print(result.profile)   # AgentIQ profiling data
# profile contains: {'total_latency_ms': 3241, 'total_tokens': 1847, 'total_cost_usd': 0.00184, 'steps': [...]}

6Profiling and Evaluation

AgentIQ's profiling and evaluation layer is what differentiates it from raw LangGraph or CrewAI:

Profiling Output

Python — Reading AgentIQ Profile Data
result = await workflow.run(input_data)

profile = result.profile
print(f"Total latency: {profile['total_latency_ms']}ms")
print(f"Total tokens: {profile['total_tokens']}")
print(f"Estimated cost: ${profile['total_cost_usd']:.4f}")

# Per-step breakdown
for step in profile['steps']:
    print(f"  [{step['name']}] {step['latency_ms']}ms "
          f"| {step['tokens']} tokens "
          f"| ${step['cost_usd']:.5f}")
# Output example:
# [researcher] 1230ms | 843 tokens | $0.00084
# [writer] 1020ms | 712 tokens | $0.00071
# [reviewer] 991ms | 292 tokens | $0.00029

NeMo Evaluator Integration

AgentIQ can pipe evaluation results directly to NeMo Evaluator for CI/CD-integrated benchmark tracking:

Python — AgentIQ + NeMo Evaluator
from aiq.eval.evaluator import run_evaluation

# Run evaluation against a golden dataset
eval_results = await run_evaluation(
    workflow=workflow,
    dataset_path="./eval_data/rag_golden.jsonl",
    metrics=["faithfulness", "answer_relevancy", "context_recall"],
    evaluator_endpoint="http://nemo-evaluator:8080",  # NeMo Evaluator service
)

print(eval_results.summary)
# {'faithfulness': 0.91, 'answer_relevancy': 0.88, 'context_recall': 0.83}
# Results automatically pushed to NeMo Evaluator dashboard
7Lab Exercise
🔬 Lab: Wrap a LangGraph Agent with AgentIQ

Objective: Take an existing LangGraph ReAct agent and add AgentIQ profiling to measure per-step cost and latency.

  1. Clone AgentIQ: git clone https://github.com/NVIDIA/AgentIQ
  2. Install: pip install aiq
  3. Build a simple LangGraph graph with 2 nodes (researcher + synthesizer)
  4. Wrap with langgraph_wrapper workflow type
  5. Run 3 test queries and compare result.profile across them
  6. Identify which node consumes the most tokens / cost
  7. Extension: Run run_evaluation() against a 5-question golden dataset and output RAGAS faithfulness scores
8Common Mistakes
MistakeFix
Confusing AgentIQ with NIMsAgentIQ = workflow toolkit; NIMs = model inference endpoints. They work together.
Thinking AgentIQ replaces LangGraphAgentIQ wraps LangGraph via langgraph_wrapper — it's additive, not a replacement
Skipping profiling in developmentProfile from day 1 — latency and cost surprises are common at scale
Not using YAML configs for agent definitionsYAML configs make agents swappable without code changes — key for prod deployments
9Exam Prep
🎯 Exam Key Points — AgentIQ (NVIDIA Platform ~7%)
  • AgentIQ = open-source NVIDIA toolkit for agentic workflow composition, profiling, and evaluation
  • NIMs = containerized inference microservices; AgentIQ calls NIM endpoints — they are complementary
  • langgraph_wrapper = wrap existing LangGraph StateGraph with AgentIQ profiling — no rewrite required
  • Profiling output: total_latency_ms, total_tokens, total_cost_usd, per-step breakdown
  • Evaluation: integrates with NeMo Evaluator via REST API for CI/CD benchmark tracking
  • Workflow types: react_agent, langgraph_wrapper, tool_calling_agent, custom
  • Config approach: YAML-driven workflow definitions for environment-agnostic deployment
