Vector Databases

How vector databases power RAG systems — HNSW indexing, approximate nearest-neighbor (ANN) search, ChromaDB vs Milvus vs pgvector, cosine vs dot-product vs Euclidean similarity, and NVIDIA NeMo Retriever integration.

1What Is a Vector Database?

A vector database stores high-dimensional embedding vectors and enables efficient similarity search — finding the K nearest vectors to a query vector in milliseconds, even across millions of documents. This is the core retrieval mechanism of every RAG pipeline.

Unlike traditional databases that match exact values (WHERE field = value), vector databases match by semantic similarity — two vectors are "close" if the texts they represent have similar meaning.

2HNSW — How Vector Search Actually Works

HNSW (Hierarchical Navigable Small World) is the dominant indexing algorithm used by ChromaDB, Milvus, Weaviate, and most production vector stores. It builds a multi-layer graph where each layer is a random subgraph of the full dataset.

Search process: Start at the top layer (sparse, long-range connections) → navigate greedily toward query → descend to lower layers (denser, short-range) → final exact search in bottom layer. Result: O(log N) search time vs O(N) for brute-force.

4Implementation
Python — ChromaDB Ingest + Search

import chromadb
from chromadb.utils import embedding_functions

# Setup
client = chromadb.PersistentClient(path="./chroma_db")
embed_fn = embedding_functions.SentenceTransformerEmbeddingFunction(
    model_name="all-MiniLM-L6-v2"  # 384-dim embeddings
)
collection = client.get_or_create_collection(
    name="ncp_aai_docs",
    metadata={"hnsw:space": "cosine"},  # distance metric
    embedding_function=embed_fn
)

# Ingest documents
documents = [
    "NVIDIA Triton uses gRPC on port 8001",
    "TensorRT-LLM supports FP8 on H100 GPUs",
    "NeMo Guardrails uses Colang language for rules"
]
collection.add(
    documents=documents,
    ids=["doc1","doc2","doc3"]
)

# Query (similarity search)
results = collection.query(
    query_texts=["What port does Triton use for gRPC?"],
    n_results=2
)
print(results["documents"])
# [["NVIDIA Triton uses gRPC on port 8001", ...]]

🎯 Exam Key Points — Knowledge Integration (10%)
  • HNSW = Hierarchical Navigable Small World = dominant vector search algorithm
  • ANN (Approximate Nearest Neighbor) vs exact KNN — ANN trades tiny accuracy for massive speed
  • Cosine similarity is standard for text embedding RAG (use dot-product when vectors are L2-normalized)
  • ChromaDB default space: cosine. Milvus default: L2 (Euclidean)
  • NeMo Retriever: multimodal extraction (tables, charts from PDFs) + NV-Embed-QA + NV-RerankQA NIMs
