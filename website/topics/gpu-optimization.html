GPU Optimization for LLM Inference

Understand how TensorRT-LLM and vLLM maximize GPU throughput via Continuous Batching and PagedAttention.

Table of Contents

  • 2 The KV Cache Bottleneck
  • 3 PagedAttention

1Throughput vs Latency

Latency: Time to first token (TTFT) or time per token. Critical for interactive chatbots.

Throughput: Total tokens generated per second across all users. Critical for Agentic workflows performing asynchronous background tasks.

Optimization techniques often sacrifice latency slightly to massively scale throughput.

2The KV Cache Bottleneck

During inference, an LLM generates tokens autoregressively (one by one). To compute Token 5, it needs to attend to Tokens 1-4. Rather than re-computing Tokens 1-4 from scratch, the Keys and Values (K & V) from the Transformer attention mechanism are cached in VRAM.

This KV Cache grows linearly with context size and batch size. Multi-agent loops sending thousands of tokens memory-starve GPUs exceptionally fast.

3PagedAttention

Historically, KV cache memory was allocated contiguously per request. If a request might need 4k tokens, 4k of contiguous memory was blocked, leading to massive memory fragmentation (wasted space) if the request only used 500 tokens.

PagedAttention solves this by partitioning the KV cache into non-contiguous "pages" (blocks of tokens), dynamically allocating them as the sequence grows. This eliminates memory fragmentation and allows the server to pack many more concurrent requests onto the GPU.

4In-flight / Continuous Batching

Standard "Static" Batching waits for all requests in a batch to finish generating their sequences before starting the next batch. If one request is 100 tokens and another is 1000, the GPU idles waiting for the 1000-token request to finish.

Continuous Batching (or In-flight Batching) evaluates at the token level. As soon as Request A finishes, Request C is dynamically inserted into the batch *at the next token generation step*, keeping GPU utilization at 100%.

5Exam Question Targets

Exam Trap: What NVIDIA library implements In-flight Batching and FP8 quantization out-of-the-box for peak inference?

Answer: TensorRT-LLM (often deployed via Triton Inference Server).
