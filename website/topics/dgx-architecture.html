1. DGX System Overview

A DGX is NVIDIA's purpose-built AI supercomputer — a rack-mountable system combining multiple high-end GPUs with a high-bandwidth NVLink fabric, fast host networking (InfiniBand), and optimized software stack (CUDA, cuDNN, NCCL, Fabric Manager).

Why DGX Matters for Certification
The NCP-AII exam (31% Systems & Bring-Up) tests your ability to bring up, configure, and verify DGX systems. You need to know which GPU is in which system, interconnect speeds, and the correct verification commands.

Exam Trap: HBM Generation vs GPU Generation
H100 uses HBM3; H200 uses HBM3e (same GPU die, more memory bandwidth). B200 also uses HBM3e but at higher capacity (192 GB). Know which GPU pairs with which HBM generation.

3. NVLink & NVSwitch Topology

Within a DGX, all GPUs connect through NVSwitch ASICs forming an all-to-all non-blocking fabric. This allows any GPU to communicate with any other at full bandwidth simultaneously — critical for model parallelism.

NVLink GenBandwidth/LinkLinks/GPUTotal GPU-to-GPU BWDGX System

# Verify NVLink topology on a DGX
nvidia-smi topo -m          # Show GPU topology matrix
nvidia-smi nvlink -s        # NVLink status per GPU
nvidia-smi nvlink -e        # NVLink error counters
nvidia-smi nvlink -c        # Clear NVLink error counters
NVSwitch Role
NVSwitches (4 in DGX A100, 4 in DGX H100) provide the crossbar fabric. Each GPU connects to all NVSwitches so every GPU-to-GPU path has equal bandwidth (non-blocking). Fabric Manager must run to activate multi-GPU NVLink.

Memory bandwidth is often the bottleneck for LLM inference. A larger HBM capacity reduces the need to offload KV-cache to host RAM, enabling larger batch sizes and longer context windows.

5. Host Networking (InfiniBand)

DGX nodes communicate across the cluster via InfiniBand NICs. Each DGX has 8 NICs — one per GPU — maximizing bandwidth for all-reduce operations in distributed training.

6. Key nvidia-smi Commands for DGX

# Basic GPU discovery
nvidia-smi                          # Summary of all GPUs
nvidia-smi -L                       # List all GPUs with UUID
nvidia-smi --query-gpu=name,memory.total,driver_version --format=csv

# GPU health and utilization
nvidia-smi dmon -s u                # Monitor utilization continuously
nvidia-smi dmon -s p                # Monitor power
nvidia-smi pmon                     # Per-process GPU usage

# Topology and connectivity
nvidia-smi topo -m                  # Topology matrix (NVLink, PCIe, SMP)
nvidia-smi topo -mp                 # P2P capability matrix
nvidia-smi topo -p2p n              # NVLink P2P matrix

# Temperature and power
nvidia-smi --query-gpu=temperature.gpu,power.draw --format=csv,noheader
nvidia-smi -pm 1                    # Enable persistent mode (recommended for DGX)

# ECC errors
nvidia-smi --query-gpu=ecc.errors.corrected.total.aggregate --format=csv
nvidia-smi --query-gpu=ecc.errors.uncorrected.total.aggregate --format=csv

7. BMC & System Management

The Baseboard Management Controller (BMC) provides out-of-band management — accessible even when the OS is down. Used for power, sensors, SEL logs, and remote console.

# ipmitool commands for DGX BMC
ipmitool sdr type Temperature       # All temperature sensors
ipmitool sdr type Fan               # Fan speeds
ipmitool sel list                   # System Event Log (last 20 entries)
ipmitool sel elist                  # Verbose SEL with event descriptions
ipmitool power status               # Current power state
ipmitool power on                   # Power on the system
ipmitool chassis status             # Chassis status summary
ipmitool lan print 1                # BMC network configuration
ipmitool fru print                  # Field Replaceable Unit info (serial numbers)
ipmitool user list 1                # BMC user accounts

# Remote BMC access
ipmitool -H  -U admin -P  power status
ipmitool -H  -U admin -P  sol activate   # Serial-over-LAN console
SEL Log Priority
Always check ipmitool sel list first when a system behaves unexpectedly. Critical events (GPU thermal shutdown, PSU failure, memory errors) are logged before the OS can report them.

8. Exam Prep & Quick Reference

Must-Know for Exam (31% domain weight)
  • DGX A100 = HBM2e, NVLink 3, 12 links, HDR IB | H100 = HBM3, NVLink 4, 18 links, NDR IB
  • H200 = same die as H100, HBM3e, 141 GB per GPU
  • Fabric Manager must be running for multi-GPU NVLink to function
  • nvidia-smi topo -m shows NVLink connectivity between all GPU pairs
  • BMC is independent of host OS — use ipmitool for hardware-level diagnostics
  • ipmitool sel list for system event log, ipmitool fru print for serial numbers
  • Persistent mode (nvidia-smi -pm 1) should always be enabled on DGX for production
