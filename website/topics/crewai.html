1What Is CrewAI?

CrewAI is an open-source Python framework for orchestrating role-based, collaborative multi-agent systems. It models real-world teams: each AI agent has a defined role, a goal, and a backstory that shapes its behavior. Agents are assigned Tasks and the Crew coordinates their execution.

Released in late 2023 and battle-tested with 10 million+ agent executions in 30 days in enterprise production, CrewAI is one of the most widely deployed multi-agent frameworks available.

💡 Mental Model

Think of CrewAI as running a consulting firm: you hire specialists (Agents), give them projects (Tasks), and the firm director (Crew) assigns work and collects results. The framework handles turn-taking, context passing, and output formatting automatically.

2Why CrewAI Exists

Single-agent architectures break down when a task requires:

  • Different expertise domains (research + coding + writing)
  • Verification and critique of generated outputs
  • Parallel workstreams that can run simultaneously
  • Long multi-step pipelines with state across tasks
  • Clear accountability — knowing which agent produced what
🎯 Exam Angle

CrewAI vs LangGraph: CrewAI is role-centric (who does what) while LangGraph is graph-centric (how execution flows). CrewAI excels at structured team workflows; LangGraph excels at complex cyclic stateful orchestration with fine-grained control.

Agent Attributes in Depth

Python — Agent Definition
from crewai import Agent
from crewai_tools import SerperDevTool

researcher = Agent(
    role="Senior Research Analyst",          # Persona label
    goal="Uncover groundbreaking AI papers",  # What the agent is optimizing for
    backstory="""You are an expert at analyzing academic papers,
    identifying key contributions, and explaining complex topics clearly.
    You always verify information from multiple sources.""",
    tools=[SerperDevTool()],                 # Tools this agent can use
    llm="nvidia/meta/llama-3.1-70b-instruct",
    verbose=True,                            # Log agent reasoning steps
    allow_delegation=False,                  # Can this agent assign subtasks?
    max_iter=5,                              # Max ReAct-style reasoning loops
    memory=True,                             # Enable agent-level memory
)

4CrewAI Flows — Stateful Pipelines

CrewAI Flows (introduced 2024) add explicit state management and conditional routing to multi-crew pipelines. A Flow is a Python class where methods are decorated with @start(), @listen(), and @router() to define execution order.

Python — CrewAI Flow with State
from crewai.flow.flow import Flow, listen, start, router
from pydantic import BaseModel

class ResearchState(BaseModel):
    topic: str = ""
    research_output: str = ""
    quality_score: float = 0.0
    requires_revision: bool = False

class ResearchFlow(Flow[ResearchState]):

    @start()
    def initialize(self):
        self.state.topic = "NVIDIA TensorRT-LLM quantization"
        print(f"Starting research on: {self.state.topic}")

    @listen(initialize)
    def run_research(self):
        # Kick off a full Crew
        result = research_crew.kickoff(
            inputs={"topic": self.state.topic}
        )
        self.state.research_output = result.raw
        # Score quality (could call another LLM here)
        self.state.quality_score = evaluate_quality(result.raw)

    @router(run_research)
    def check_quality(self):
        if self.state.quality_score >= 0.8:
            return "approved"
        else:
            self.state.requires_revision = True
            return "revision_needed"

    @listen("approved")
    def publish(self):
        print("Research approved! Publishing...")
        save_to_database(self.state.research_output)

    @listen("revision_needed")
    def revise(self):
        print("Quality too low. Triggering revision crew...")
        revision_crew.kickoff(
            inputs={"draft": self.state.research_output}
        )

# Run the flow
flow = ResearchFlow()
flow.kickoff()
💡 Flows vs Crews

Crew = one team working on a set of related tasks. Flow = orchestration of multiple Crews with shared state, routing, and branching logic. Use Flows when you need to chain multiple Crews together or add conditional logic between them.

5Full Python Example — Research & Writing Pipeline
Python — Complete CrewAI Pipeline
from crewai import Agent, Task, Crew, Process
from crewai_tools import SerperDevTool, FileReadTool
from langchain_nvidia_ai_endpoints import ChatNVIDIA

# 1. Configure LLM (NVIDIA NIM)
llm = ChatNVIDIA(
    model="meta/llama-3.1-70b-instruct",
    base_url="https://integrate.api.nvidia.com/v1",
    temperature=0.1,
)

# 2. Define Agents
researcher = Agent(
    role="Research Specialist",
    goal="Find accurate, up-to-date information on the given topic",
    backstory="Expert researcher who verifies facts across multiple sources.",
    tools=[SerperDevTool()],
    llm=llm,
    verbose=True,
    allow_delegation=False,
)

writer = Agent(
    role="Technical Writer",
    goal="Transform research into clear, structured documentation",
    backstory="Experienced technical writer who creates developer-friendly content.",
    tools=[],
    llm=llm,
    verbose=True,
    allow_delegation=False,
)

reviewer = Agent(
    role="Quality Reviewer",
    goal="Ensure technical accuracy and completeness of written content",
    backstory="Senior engineer who catches errors and identifies gaps.",
    tools=[],
    llm=llm,
    verbose=True,
    allow_delegation=True,   # Can delegate back to researcher if gaps found
)

# 3. Define Tasks (sequential with context chaining)
research_task = Task(
    description="Research NVIDIA TensorRT-LLM quantization methods. Include benchmarks.",
    expected_output="Structured research with quantization methods, benchmarks, and citations.",
    agent=researcher,
)

writing_task = Task(
    description="""Using the research provided, write a developer guide on TensorRT-LLM quantization.
    Include code examples for INT4 AWQ and FP8. Audience: ML engineers.""",
    expected_output="A complete developer guide with code examples, tables, and clear explanations.",
    agent=writer,
    context=[research_task],    # Writing task receives research output as context
)

review_task = Task(
    description="""Review the developer guide for: (1) Technical accuracy, (2) Code correctness,
    (3) Missing information. Provide specific improvement suggestions.""",
    expected_output="Detailed review with specific issues listed and severity (critical/minor).",
    agent=reviewer,
    context=[research_task, writing_task],
    output_file="review_output.md",
)

# 4. Create and Run Crew
crew = Crew(
    agents=[researcher, writer, reviewer],
    tasks=[research_task, writing_task, review_task],
    process=Process.sequential,
    verbose=True,
    memory=True,              # Enable cross-task memory
    embedder={
        "provider": "nvidia",
        "config": {"model": "NV-Embed-QA"}
    }
)

result = crew.kickoff()
print(result.raw)             # Final output (last task's result)
print(crew.usage_metrics)     # Token counts per task per agent

Objective: Create a CrewAI pipeline that researches a topic, writes a report, and reviews it for quality.

  1. Install: pip install crewai crewai-tools langchain-nvidia-ai-endpoints
  2. Set NVIDIA_API_KEY environment variable
  3. Create a Researcher agent with SerperDevTool and llama-3.1-70b via NVIDIA NIMs
  4. Create a Writer agent (no tools) that takes research context
  5. Create a Reviewer agent with allow_delegation=True
  6. Run with Process.sequential and observe context passing between tasks
  7. Extension: Add a Flow with a quality router — if Reviewer score < 0.7, trigger a revision task

Expected output: A multi-section research report with reviewer feedback demonstrating context chaining between tasks.

8Common Mistakes
MistakeImpactFix
Setting allow_delegation=True on all agentsAgents delegate instead of executing, infinite loopsOnly the orchestrator/manager needs delegation
Vague expected_outputAgents produce inconsistent formatsDescribe exact structure: sections, format, length
No context chain between dependent tasksWriter doesn't see researcher outputPass upstream tasks in context=[]
Same LLM for all agentsExpensive; overkill for simple rolesUse smaller models for summarizers, larger for critics
Ignoring usage_metricsNo visibility into token costs per agentLog crew.usage_metrics after every kickoff
Confusing Process.hierarchical with Flow routingWrong architecture for the taskhierarchical = dynamic delegation within one crew; Flow = routing between multiple crews
9Exam Prep
🎯 Exam Key Points — CrewAI (Agent Architecture 15%)
  • Primitives: Agent (role + goal + backstory + tools) → Task (description + expected_output + agent + context) → Crew (agents + tasks + process)
  • Process.sequential: tasks run in order; each task receives previous task output via context
  • Process.hierarchical: manager agent dynamically assigns tasks to worker agents based on capability
  • allow_delegation=True: agent can sub-delegate tasks to other agents in the crew
  • CrewAI Flows: use @start(), @listen(), @router() decorators; share state via Pydantic model
  • vs LangGraph: CrewAI = role-centric team workflow; LangGraph = graph-centric cyclic state machine
  • vs AutoGen: CrewAI = structured task assignment; AutoGen = conversational turn-taking
  • memory=True: enables cross-task and cross-session memory using embeddings
  • Scale: 10M+ agent executions in 30 days in production — production-grade, not demo-only
  • NVIDIA integration: via langchain_nvidia_ai_endpoints.ChatNVIDIA or OpenAI-compatible NIM endpoint
