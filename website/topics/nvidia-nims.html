NVIDIA NIMs

NVIDIA Inference Microservices — pre-packaged, optimized LLM containers with OpenAI-compatible API. The fastest path from model to production agent. Includes AgentIQ toolkit for agentic workflows.

1What Are NIMs?

NVIDIA Inference Microservices (NIMs) are pre-built Docker containers that bundle a model, TensorRT-LLM engine, Triton Inference Server, and an OpenAI-compatible REST API. You get production-optimized inference without manually configuring Triton or TensorRT-LLM.

API endpoint: https://integrate.api.nvidia.com/v1 (cloud) or http://localhost:8000/v1 (local container). Both use the same OpenAI API format.

Python — Call NVIDIA NIM

from openai import OpenAI

client = OpenAI(
    base_url="https://integrate.api.nvidia.com/v1",
    api_key="nvapi-xxxxxxxxxxxxxx"   # from build.nvidia.com
)

response = client.chat.completions.create(
    model="meta/llama-3.1-70b-instruct",
    messages=[
        {"role": "system", "content": "You are a helpful AI agent."},
        {"role": "user", "content": "Explain paged KV cache in 2 sentences."}
    ],
    temperature=0.2,
    max_tokens=256
)
print(response.choices[0].message.content)
2Why NIMs — The Value Proposition

Deploying optimized LLM inference manually requires: model conversion → TRT-LLM build → Triton config → Docker setup → scaling config. NIMs collapse this into a single docker pull.

  • OpenAI-compatible — drop-in replacement: change base_url, keep all LangChain/OpenAI code
  • Available everywhere — AWS, Azure AI Foundry, GCP, Oracle, on-prem, local GPU
  • Llama Nemotron + more — NVIDIA's own optimized models available via NGC registry
  • AgentIQ integration — direct plugin support for agentic tool calling workflows
3AgentIQ — NVIDIA's Agent Toolkit

AgentIQ is NVIDIA's open-source toolkit for building, evaluating, and optimizing agentic workflows that use NIMs. It provides:

  • Composable workflow primitives (tools, agents, pipelines)
  • Built-in evaluation framework for agentic systems
  • Performance profiling and optimization suggestions
  • Plugin system: integrates with LangGraph, LangChain, LlamaIndex
  • Supports langgraph_wrapper workflow type for existing LangGraph agents
Python — LangChain + NVIDIA NIM

from langchain_nvidia_ai_endpoints import ChatNVIDIA

llm = ChatNVIDIA(
    model="meta/llama-3.1-70b-instruct",
    api_key="nvapi-xxxx",
    base_url="https://integrate.api.nvidia.com/v1"
)

# Works with all LangChain chains, agents, and LangGraph graphs
response = llm.invoke("Explain HNSW indexing for vector databases.")

🎯 Exam Key Points — NVIDIA Platform (7%)
  • NIM API endpoint: integrate.api.nvidia.com/v1 (OpenAI-compatible)
  • Free API key: build.nvidia.com
  • AgentIQ = NVIDIA's open-source agentic optimization toolkit
  • NeMo microservices GA: April 23, 2025 (Customizer, Evaluator, Guardrails, Retriever, Curator)
  • langgraph_wrapper = how AgentIQ integrates existing LangGraph agents with minimal code changes
  • NIMs are available on AWS, Azure, GCP, Oracle (GTC 2025 announcement)
❌ Confusing NIMs vs NeMo Guardrails

NIMs = inference containers (model serving). NeMo Guardrails = safety layer (Colang rails). They are separate tools that work together.
