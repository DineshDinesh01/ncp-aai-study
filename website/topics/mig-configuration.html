MIG Configuration

Multi-Instance GPU partitioning — isolate a single A100 or H100 into up to 7 independent GPU instances with guaranteed memory and compute bandwidth.

1. What Is MIG?

Multi-Instance GPU (MIG) is a GPU partitioning technology available on A100 (80 GB / 40 GB), H100 SXM, and H100 PCIe. It allows one physical GPU to be split into up to 7 isolated instances, each with dedicated memory bandwidth, L2 cache, and compute resources — hardware-enforced isolation, not virtualization.

MIG vs vGPU
MIG is hardware partitioning — the GPU slices are physically separate at the chip level. vGPU is software virtualization sharing the full GPU via a hypervisor. MIG provides better isolation guarantees and is used in HPC/cloud multi-tenant scenarios.

2. MIG Architecture (GI & CI)

MIG has two levels of partitioning:

TermFull NameDescription
GIGPU InstanceTop-level partition — dedicated memory slices and SM partitions. Created first.
CICompute InstanceSub-partition within a GI — defines which SMs within the GI are usable. Created second inside a GI.

A workload runs on a MIG device only when both a GI and a CI exist. The combined (GI+CI) is exposed as a MIG device (e.g., MIG-GPU-UUID/4/0).

3. MIG Profiles on A100/H100

Profiles define how many GPU Memory Slices (GMS) and SM partitions each instance gets. The A100 80 GB has 8 GMS (1 GMS = 10 GB).

Profile NameMemory (A100 80GB)Memory (H100 80GB)Max Instances per GPUSMs
1g.10gb10 GB10 GB71/7
2g.20gb20 GB20 GB32/7
3g.40gb40 GB40 GB23/7
4g.40gb40 GB40 GB14/7
7g.80gb80 GB (full GPU)80 GB17/7
Profile Naming Format
Format: {compute_slices}g.{memory_gb}gb. The number before "g" is compute slice count (not GPU count). 1g.10gb = 1 compute slice, 10 GB memory. H100 SXM has additional profiles like 1g.10gb+me with Media Extensions.

4. Enabling & Disabling MIG

# Enable MIG mode on GPU 0 (requires root, terminates running processes)
sudo nvidia-smi -i 0 -mig 1

# Disable MIG mode on GPU 0
sudo nvidia-smi -i 0 -mig 0

# Check MIG mode status for all GPUs
nvidia-smi --query-gpu=index,name,mig.mode.current --format=csv,noheader

# Enable MIG on all GPUs in the system
sudo nvidia-smi -mig 1

# IMPORTANT: Reboot may be required after enabling MIG on some systems
# Check: nvidia-smi returns "Pending Enable" → reboot needed
Enabling MIG Terminates All GPU Processes
Enabling or disabling MIG mode kills all CUDA processes on that GPU. Always drain workloads from Slurm partitions before enabling MIG in production.

5. Creating GI and CI Instances

# Step 1 — List available GI profiles on GPU 0
nvidia-smi mig -lgip -i 0

# Step 2 — Create a GPU Instance using profile ID
# Profile IDs are listed in nvidia-smi mig -lgip output
nvidia-smi mig -cgi 9,9,9,9,9,9,9 -i 0    # Seven 1g.10gb GIs
nvidia-smi mig -cgi 5,5 -i 0               # Two 3g.40gb GIs
nvidia-smi mig -cgi 14 -i 0               # One 7g.80gb GI (full GPU as MIG)

# Step 3 — List created GIs
nvidia-smi mig -lgi -i 0

# Step 4 — Create Compute Instances within each GI
nvidia-smi mig -cci -i 0 -gi 0    # Create default CI in GI 0
nvidia-smi mig -cci -i 0          # Create default CIs in all GIs

# Step 5 — List all MIG devices (GI+CI combined)
nvidia-smi mig -lme

# Delete all instances and start fresh
sudo nvidia-smi mig -dci && sudo nvidia-smi mig -dgi -i 0

6. Verification Commands

# List all MIG devices with UUIDs
nvidia-smi -L

# Expected output for 7x 1g.10gb on GPU 0:
# GPU 0: A100 80GB SXM4 (UUID: GPU-xxxx)
#   MIG 1g.10gb     Device  0: (UUID: MIG-GPU-xxxx/1/0)
#   MIG 1g.10gb     Device  1: (UUID: MIG-GPU-xxxx/2/0)
#   ...

# Show detailed MIG topology
nvidia-smi mig -lme -i 0

# Check CUDA_VISIBLE_DEVICES for MIG (use full MIG UUID)
export CUDA_VISIBLE_DEVICES=MIG-GPU-xxxx/1/0
python3 -c "import torch; print(torch.cuda.get_device_name(0))"

# Query MIG-specific metrics
nvidia-smi --query-gpu=index,mig.mode.current,gpu_uuid --format=csv

7. MIG with DCGM

DCGM (Data Center GPU Manager) supports MIG instances natively. Each MIG device gets its own DCGM entity and can be monitored/alerted independently.

# Discover MIG entities in DCGM
dcgmi discovery -l          # Lists all GPUs and MIG instances

# Field watch on a MIG instance entity ID
dcgmi dmon -e 203,204,1001 -d 1000   # SM util, memory util, MIG mode per entity

# Create DCGM group with only MIG instances
dcgmi group -c mig_group --default
dcgmi group -a -g mig_group -i MIG:GPU-xxxx/1/0

8. Exam Prep & Quick Reference

Must-Know for Exam
  • MIG is available on: A100, H100, H200 — NOT on A10, A30, T4, or consumer GPUs
  • Maximum 7 MIG instances per A100/H100 GPU (using 1g.10gb profile)
  • Creating GI comes before creating CI — both needed to use the MIG device
  • Enable MIG: nvidia-smi -i 0 -mig 1, then reboot may be required
  • MIG devices in CUDA_VISIBLE_DEVICES use the full UUID format
  • MIG does NOT support NVLink between instances — each instance is isolated
  • Profile format: {compute_slices}g.{memory_gb}gb
