Tokenization

How LLMs convert raw text into numbers — BPE, WordPiece, SentencePiece, vocab size tradeoffs, and NVIDIA NeMo tokenizer configuration for production agent pipelines.

1What Is Tokenization?

Tokenization is the process of converting raw text into a sequence of integer token IDs that a language model can process. Every LLM — whether GPT-4, Llama-3, or a NeMo-fine-tuned model — only sees token IDs, never raw text.

A tokenizer consists of two components: a vocabulary (a fixed mapping of text pieces → integer IDs) and an encoding algorithm (the rules for splitting text into those pieces).

2Why Does Tokenization Exist?

Neural networks operate on fixed-size numerical vectors, not strings. Tokenization bridges the gap between human-readable text and model-readable numbers. The efficiency of tokenization directly determines:

  • Context window usage — more tokens per word = less text fits in context
  • Inference cost — billing is per token in NIM/API calls
  • Multilingual performance — poor tokenization of non-English text wastes tokens
  • Code/math accuracy — tokenization of special characters affects reasoning
3Three Main Algorithms

Byte-Pair Encoding (BPE) — Used by GPT-2, GPT-4, Llama, most NVIDIA NIMs. Starts with character-level vocabulary, then iteratively merges the most frequent adjacent pair into a single token, until vocab size is reached.

BPE Algorithm

# Simplified BPE merge step
vocab = {("h","e","l","l","o"): 5, ("w","o","r","l","d"): 3}
# Most frequent pair: ("l","l") → merge → "ll"
# Repeat until vocab_size reached

WordPiece — Used by BERT, DistilBERT. Similar to BPE but merges based on likelihood ratio (maximize training data log-likelihood) rather than raw frequency. Uses ## prefix for subword continuations.

SentencePiece (Unigram LM) — Used by Llama, T5, ALBERT. Treats tokenization as a probabilistic model. Trains directly on raw bytes (no pre-tokenization) — handles any language including Japanese, Chinese, Arabic without whitespace assumptions.

4Where Tokenization Fits in an Agent System

In an agentic pipeline, tokenization happens at two critical points:

  1. Input encoding — User message + system prompt + tool outputs are tokenized before LLM call
  2. Output decoding — LLM generates token IDs; tokenizer decodes them back to text
⚠️ Common Agent Bug

Agents fail when tool outputs push the total token count beyond the model's context window. Always count tokens before adding to messages. Use tiktoken (OpenAI) or NeMo's tokenizer to count precisely.

5Implementation Examples
Python — Token Counting for Agents

import tiktoken

enc = tiktoken.encoding_for_model("gpt-4o")

def count_tokens(messages: list[dict]) -> int:
    """Count tokens in a ChatML message list."""
    total = 0
    for msg in messages:
        total += 4  # per-message overhead
        for key, value in msg.items():
            total += len(enc.encode(str(value)))
    total += 2  # reply priming
    return total

messages = [
    {"role": "system", "content": "You are a helpful AI agent."},
    {"role": "user", "content": "Summarize the following document: " + "word " * 3000}
]
print(f"Token count: {count_tokens(messages)}")
# → Token count: 3024 (approaching gpt-4o 128k limit? no, fine here)
Python — HuggingFace Tokenizer

from transformers import AutoTokenizer

# Load tokenizer for Llama-3 (uses SentencePiece)
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Meta-Llama-3-8B")

text = "The NVIDIA NIM API is OpenAI-compatible."
tokens = tokenizer.encode(text)
decoded = tokenizer.decode(tokens)

print(f"Token IDs: {tokens}")
print(f"Token strings: {[tokenizer.decode([t]) for t in tokens]}")
print(f"Token count: {len(tokens)}")
# Token count: 11 (Llama-3 is efficient with technical terms)
6NVIDIA Ecosystem Mapping

Tokenization is configured in NeMo Customizer when fine-tuning models. The tokenizer must exactly match the base model's tokenizer — mismatches cause silent errors during SFT.

YAML — NeMo Customizer SFT Config

# nemo_customizer_sft.yaml
model:
  name: meta/llama-3.1-8b-instruct
  tokenizer:
    type: sentencepiece       # must match base model
    vocab_file: null          # auto-loaded from model checkpoint
  
data:
  train_ds:
    file_path: /data/train.jsonl
    max_seq_length: 4096      # token limit — controls GPU memory
    add_bos: true             # add beginning-of-sequence token
    add_eos: true             # add end-of-sequence token

When calling NVIDIA NIMs, the API handles tokenization server-side. However, for context window management in your agent, use the same tokenizer locally to count tokens before sending.

8Common Mistakes
❌ Mistake 1 — Not Counting Tokens Before API Calls

Sending tool outputs to an LLM without checking token count causes truncation or API errors. Always count before adding to messages.

❌ Mistake 2 — Assuming 1 Word = 1 Token

English averages ~1.3 tokens/word. Code, numbers, and special characters can be 3–5 tokens/word. Never estimate — always count programmatically.

❌ Mistake 3 — Using Wrong Tokenizer for Token Counting

If you use tiktoken (GPT-4) to count tokens for a Llama-3 NIM call, the count will be wrong. Match tokenizer to model.
